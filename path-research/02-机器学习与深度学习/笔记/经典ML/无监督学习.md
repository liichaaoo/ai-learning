# 无监督学习（Unsupervised Learning）详解

## 一、什么是无监督学习

**无监督学习**：训练数据**只有输入 \(x\)，没有标签 \(y\)**，模型需要自己发现数据中的内在结构、规律或分布。

> 类比：给学生一堆没有答案的练习题，让他自己把相似的归成一类、找出哪些题目本质相同——没人告诉他"对错"。

形式化：
- 训练集：\(D = \{x_1, x_2, \dots, x_n\}\)（**没有 \(y\)**）
- 学习目标：发现数据的**结构 / 分布 / 表征**

---

## 二、和监督学习的本质区别

| 维度 | 监督学习 | 无监督学习 |
|------|---------|-----------|
| 数据 | \((x, y)\) 有标签 | 只有 \(x\) |
| 目标 | 学习 \(x \to y\) 映射 | 发现数据内在结构 |
| 评估 | 客观（准确率、MSE） | **主观/间接**（轮廓系数、人工判断） |
| 任务 | 分类、回归 | 聚类、降维、密度估计、异常检测 |
| 典型场景 | 预测 | 探索、压缩、预处理 |
| 数据成本 | 高（要标注） | 低（原始数据即可） |

> **关键洞察**：现实世界 95% 的数据都是无标签的，所以无监督学习 + 自监督学习是 AI 的未来方向（BERT、GPT、对比学习都是它的延伸）。

---

## 三、四大核心任务

### 1. 聚类（Clustering）
把相似样本分到同一组。
- 用户分群、新闻主题归类、图像分割

### 2. 降维（Dimensionality Reduction）
把高维数据压缩到低维，保留主要信息。
- 数据可视化、特征压缩、去噪、加速训练

### 3. 密度估计（Density Estimation）
估计数据的概率分布 \(p(x)\)。
- 异常检测、生成模型基础

### 4. 异常检测（Anomaly Detection）
找出不符合主流模式的"离群点"。
- 信用卡欺诈、设备故障、网络入侵检测

### 5. 关联规则挖掘（Association Rule Learning）
发现"买了 A 的人也常买 B"这类模式。
- 购物篮分析（啤酒和尿布）、推荐系统

## 四、聚类算法详解（最核心）

### 1. K-Means（最经典）

**思想**：把数据划分成 K 个簇，使每个样本到所属簇中心的距离最小。

**目标函数**：
\[
J = \sum_{i=1}^{K} \sum_{x \in C_i} \|x - \mu_i\|^2
\]

**算法步骤**：
1. 随机初始化 K 个簇中心 \(\mu_1, \dots, \mu_K\)
2. **分配**：每个样本归到最近的簇中心
3. **更新**：重新计算每个簇的中心（均值）
4. 重复 2-3，直到中心不再变化

**优点**：简单、高效、可扩展
**缺点**：
- 必须**预先指定 K**
- 对**初始中心敏感**（解决：K-Means++ 初始化）
- 只适合**凸形球状簇**，不适合环形/月牙形
- 对**异常值敏感**（均值会被拉偏）

**如何选 K？**
- **肘部法则（Elbow Method）**：画 K-SSE 曲线，找拐点
- **轮廓系数（Silhouette Score）**：取最大值对应的 K
- **业务经验**：用户分群常分 3~5 类

### 2. 层次聚类（Hierarchical Clustering）

**思想**：构造一棵聚类树（树状图 Dendrogram）。
- **凝聚式（自底向上）**：每个点是一个簇，逐步合并最近的两簇
- **分裂式（自顶向下）**：从一个大簇逐步分裂

**簇间距离度量**：
- Single Linkage（最小距离）—— 链状效应
- Complete Linkage（最大距离）—— 紧凑簇
- Average Linkage（平均距离）
- Ward（方差最小化）—— 最常用

**优点**：不用预先指定 K，可视化清晰
**缺点**：复杂度 \(O(n^3)\) 或 \(O(n^2 \log n)\)，不适合大数据

### 3. DBSCAN（基于密度）

**思想**：高密度区域是簇，低密度区域是噪声。

**两个核心参数**：
- \(\varepsilon\)（eps）：邻域半径
- MinPts：成为核心点所需的最少邻居数

**点的三种类型**：
- **核心点**：邻域内点数 ≥ MinPts
- **边界点**：在某核心点邻域内，但自己不是核心点
- **噪声点**：既不是核心点也不是边界点

**优点**：
- **不需要指定簇数 K**
- 可发现**任意形状**的簇
- 自动识别**异常点**

**缺点**：对参数敏感，**密度不均**时效果差（解决：HDBSCAN）

### 4. GMM（高斯混合模型）

**思想**：假设数据由 K 个高斯分布混合产生，用 EM 算法估计参数。

\[
p(x) = \sum_{k=1}^{K} \pi_k \mathcal{N}(x \mid \mu_k, \Sigma_k)
\]

**EM 算法**：
- **E 步**：计算每个样本属于每个高斯的概率（软分配）
- **M 步**：用加权样本更新 \(\mu_k, \Sigma_k, \pi_k\)

**vs K-Means**：
- K-Means 是**硬分配**（每个点只属于一个簇）
- GMM 是**软分配**（给出每个点属于各簇的概率）
- GMM 可建模**椭圆形**簇，K-Means 只能球形

### 5. 谱聚类（Spectral Clustering）

把聚类转化为图划分问题，对相似度矩阵的拉普拉斯矩阵做特征分解后再聚类。
适合**非凸**形状、流形数据。

### 聚类算法对比表

| 算法 | 是否需要 K | 簇形状 | 处理噪声 | 复杂度 | 适用场景 |
|------|-----------|--------|---------|--------|---------|
| K-Means | ✅ 需要 | 球形 | ❌ | \(O(nKt)\) | 一般用途、快速 |
| 层次聚类 | ❌ | 任意 | ❌ | \(O(n^2)\) | 小数据、可视化 |
| DBSCAN | ❌ | 任意 | ✅ | \(O(n \log n)\) | 异常检测、空间数据 |
| GMM | ✅ 需要 | 椭圆 | ❌ | 中等 | 软聚类、概率建模 |
| 谱聚类 | ✅ 需要 | 任意 | 一般 | \(O(n^3)\) | 流形/图结构数据 |

## 五、降维算法详解

### 1. PCA（主成分分析）—— 最经典的线性降维

**思想**：找到一组正交方向（主成分），使数据投影后**方差最大**（保留信息最多）。

**算法步骤**：
1. 数据中心化：\(x \leftarrow x - \bar{x}\)
2. 计算协方差矩阵 \(\Sigma = \frac{1}{n} X^T X\)
3. 对 \(\Sigma\) 做特征分解，得特征值 \(\lambda_i\) 和特征向量 \(v_i\)
4. 取最大的 k 个特征值对应的特征向量构成投影矩阵 \(W\)
5. 降维：\(z = W^T x\)

**如何选 k？**
- 累计方差贡献率达到 85%~95%

**优缺点**：
- ✅ 简单、可解释、有解析解
- ❌ 只能捕捉**线性**结构，对非线性流形无效

### 2. t-SNE（可视化神器）

**思想**：保持高维空间中**邻近关系**，将数据映射到 2D/3D。
- 高维用高斯核、低维用 t 分布（重尾，缓解拥挤问题）

**特点**：
- ✅ 可视化效果惊艳（簇分得很开）
- ❌ 计算慢、结果**随机**、距离不可解释、不能用于新样本投影
- ⚠️ **只用来可视化，不要拿来做特征**

### 3. UMAP（t-SNE 升级版）

- 比 t-SNE **快**几十倍
- 更好地保持**全局结构**
- 可以投影新样本
- 当前数据可视化首选

### 4. 自编码器（Autoencoder）—— 神经网络降维

**结构**：Encoder → 低维隐空间 → Decoder
**训练目标**：让重构 \(\hat{x}\) 尽量接近 \(x\)（重构误差最小）

变体：
- **稀疏自编码器** Sparse AE
- **去噪自编码器** Denoising AE（输入加噪声）
- **变分自编码器** VAE（生成模型）
- **对比学习** SimCLR/MoCo（自监督）

### 5. LDA（线性判别分析，注意：是有监督降维）

虽然叫"判别分析"，但**它需要标签**，不是无监督方法。这里特别提示一下避免混淆。

### 降维算法对比

| 算法 | 线性/非线性 | 用途 | 速度 | 备注 |
|------|------------|------|------|------|
| PCA | 线性 | 通用降维、去噪 | 快 | 首选基线 |
| t-SNE | 非线性 | 可视化 | 慢 | 只可视化 |
| UMAP | 非线性 | 可视化、降维 | 较快 | 现代首选 |
| Autoencoder | 非线性 | 特征学习 | 中等 | 灵活、需调参 |

## 六、异常检测算法

| 算法 | 思想 |
|------|------|
| **Isolation Forest** | 用随机树"孤立"异常点（路径短即异常）—— 工业界首选 |
| **One-Class SVM** | 学习正常数据的边界 |
| **LOF（局部离群因子）** | 基于局部密度比较 |
| **Autoencoder** | 重构误差大的就是异常 |
| **DBSCAN** | 噪声点即异常 |

## 七、评估指标（无监督的难点）

无监督**没有"标准答案"**，评估只能间接做：

### 1. 内部指标（不需要真实标签）
- **轮廓系数 Silhouette**：\([-1, 1]\)，越大越好
  \[
  s(i) = \frac{b(i) - a(i)}{\max(a(i), b(i))}
  \]
  \(a(i)\)：到同簇平均距离；\(b(i)\)：到最近异簇平均距离
- **Calinski-Harabasz 指数**：簇间方差/簇内方差，越大越好
- **Davies-Bouldin 指数**：越小越好

### 2. 外部指标（如果恰好有真实标签作为参考）
- **ARI（调整兰德指数）**
- **NMI（归一化互信息）**

### 3. 业务评估
- 聚类结果交给业务方解读：能不能命名？能不能用于运营？
- 这通常比数学指标更重要。

## 八、最小可运行代码示例

### K-Means + PCA 可视化

```python
import numpy as np
import matplotlib.pyplot as plt
from sklearn.datasets import load_iris
from sklearn.cluster import KMeans
from sklearn.decomposition import PCA
from sklearn.preprocessing import StandardScaler
from sklearn.metrics import silhouette_score

# 1. 加载数据（不用标签）
X, y_true = load_iris(return_X_y=True)

# 2. 标准化（K-Means、PCA 都对量纲敏感）
X_std = StandardScaler().fit_transform(X)

# 3. K-Means 聚类
kmeans = KMeans(n_clusters=3, n_init=10, random_state=42)
labels = kmeans.fit_predict(X_std)

# 4. 评估（轮廓系数）
print("Silhouette:", silhouette_score(X_std, labels))

# 5. PCA 降到 2D 可视化
X_2d = PCA(n_components=2).fit_transform(X_std)
plt.scatter(X_2d[:, 0], X_2d[:, 1], c=labels, cmap='viridis')
plt.title("K-Means on Iris (PCA-2D)")
plt.show()
```

### 用肘部法则选 K

```python
sse = []
for k in range(1, 10):
    km = KMeans(n_clusters=k, n_init=10, random_state=42).fit(X_std)
    sse.append(km.inertia_)
plt.plot(range(1, 10), sse, marker='o')
plt.xlabel("K"); plt.ylabel("SSE"); plt.title("Elbow Method")
plt.show()
```

## 十、实战注意事项（踩坑指南）

1. **特征必须标准化**：K-Means、PCA 都基于距离/方差，量纲不一致会被大数值特征主导
2. **K-Means 多跑几次**：初始化随机，用 `n_init=10` 取最优
3. **类别特征要编码**：K-Means 不能直接用 One-Hot（高维稀疏），考虑 K-Modes
4. **大数据用 MiniBatchKMeans**：节省内存
5. **t-SNE 不要用作特征**：每次结果不一样，距离也不可比
6. **聚类不是万能的**：先问"我真的需要聚类吗？"很多业务问题用规则就够了
7. **结果一定要可解释**：聚类完要看每个簇的特征均值/分布，给簇命名

## 十一、常见面试题

1. K-Means 为什么对初始中心敏感？K-Means++ 怎么改进的？
2. K-Means 和 GMM 的区别和联系？（提示：K-Means 是 GMM 协方差为 \(\sigma^2 I\) 且硬分配的特例）
3. 为什么 PCA 要中心化数据？不中心化会怎样？
4. PCA 和 LDA 的区别？（无/有监督；最大方差 vs 最大可分性）
5. DBSCAN 怎么选 \(\varepsilon\) 和 MinPts？
6. 聚类效果不好怎么办？（特征工程、换算法、调参、增加领域信息）
7. 高维数据为什么聚类难？（维度诅咒：距离趋同）

## 十二、学习路线建议

1. **先吃透 K-Means 和 PCA**（手推数学）—— 90% 场景够用
2. **DBSCAN + 层次聚类**（理解非球形和层次结构）
3. **GMM + EM 算法**（连接到概率视角）
4. **t-SNE / UMAP**（实战可视化）
5. **Autoencoder**（衔接深度学习）
6. **自监督学习 / 对比学习**（前沿，连接 NLP/CV 大模型）
