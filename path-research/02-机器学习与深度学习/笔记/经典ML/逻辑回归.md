# 逻辑回归（Logistic Regression）详解

> 名字带"回归"、本质是**分类**。是工业界最经典的二分类基线（CTR 预估、风控、医疗诊断），也是理解 **Sigmoid / 交叉熵 / 极大似然 / GLM** 四大概念的最好入口。
> 配套阅读：[线性回归.md](./线性回归.md)、[监督学习.md](./监督学习.md)

---

## 一、为什么需要逻辑回归

线性回归输出 \(z = \boldsymbol{w}^\top \boldsymbol{x} + b \in (-\infty, +\infty)\)，但分类任务要的是**概率**或**类别**。直接拿线性回归 + 阈值做分类有 3 个致命问题：

| 问题 | 说明 |
|------|------|
| **值域无界** | 概率必须在 \([0,1]\)，线性回归会预测出 \(-3\)、\(2.7\) 这种无意义的值 |
| **对异常值敏感** | 一个远离边界的正样本会把"决策超平面"拽偏 |
| **MSE + 分类是非凸的** | 难以全局优化（详见第四章） |

**逻辑回归的关键一步**：把线性输出 \(z\) 通过 **Sigmoid** 函数压到 \((0,1)\)，解释成"属于正类的概率"。

---

## 二、Sigmoid 函数

\[
\sigma(z) = \frac{1}{1 + e^{-z}}
\]

### 性质

| 性质 | 说明 |
|------|------|
| 值域 | \((0, 1)\) |
| \(\sigma(0)\) | \(0.5\)（决策边界） |
| 极限 | \(z \to +\infty,\ \sigma \to 1\)；\(z \to -\infty,\ \sigma \to 0\) |
| **导数极简** | \(\sigma'(z) = \sigma(z)(1-\sigma(z))\) |
| 单调 | 严格单调递增 |
| 对称 | \(\sigma(-z) = 1 - \sigma(z)\) |

> 推导导数（面试考过）：\(\sigma(z) = \frac{1}{1+e^{-z}}\)，对 \(z\) 求导得 \(\frac{e^{-z}}{(1+e^{-z})^2} = \sigma(z) \cdot \frac{e^{-z}}{1+e^{-z}} = \sigma(z)(1-\sigma(z))\)。

```
       σ(z)
        1 |                ___---
            |          ___---
          0.5|.......___-
            |    ___-       
            |___-           
        0 +----------------- z
            -∞       0       +∞
```

### Sigmoid 不是凭空选的

来自**伯努利分布的指数族形式 + 对数几率链接函数**（见第八章 GLM 视角），是数学上最自然的选择，并非"随便挑了一个 S 形函数"。

---

## 三、模型形式

### 二分类模型

\[
P(y=1 \mid \boldsymbol{x}) = \sigma(\boldsymbol{w}^\top \boldsymbol{x} + b) = \hat{y}
\]
\[
P(y=0 \mid \boldsymbol{x}) = 1 - \hat{y}
\]

可以合并写成（伯努利分布）：
\[
P(y \mid \boldsymbol{x}) = \hat{y}^{y} \cdot (1 - \hat{y})^{1-y},\quad y \in \{0, 1\}
\]

### 决策规则

\[
\hat{y}_{\text{cls}} = \begin{cases} 1, & \hat{y} \geq 0.5 \\ 0, & \hat{y} < 0.5 \end{cases}
\quad\Longleftrightarrow\quad \boldsymbol{w}^\top \boldsymbol{x} + b \gtreqless 0
\]

> **决策边界是线性的**——这点常被忽视。逻辑回归本质是**线性分类器**，曲线的是"概率值"而不是"决策面"。

阈值 0.5 不是必须，业务上常根据 PR/ROC 曲线调阈值（如风控倾向高 Recall，会下调阈值）。

### 举例：吴恩达课程经典图解

设 \(w_1 = 1,\ w_2 = 1,\ b = -3\)，则决策边界 \(z = 0\) 即：

\[
x_1 + x_2 - 3 = 0 \quad\Longrightarrow\quad x_1 + x_2 = 3
\]

这是一条从 \((3, 0)\) 到 \((0, 3)\) 的直线，恰好把左下角的圆圈（y=0）和右上角的叉叉（y=1）分开。

```
x_2
 3 ●
   |  \
 2 |    \           ✕ ✕ ✕     ← x1+x2 > 3 → z>0 → ŷ≥0.5 → 预测 1
   |      \         ✕ ✕
 1 | ○ ○    \  ✕ ✕
   | ○ ○ ○    \  ✕
 0 +———————————●———— x_1
   0   1   2   3
   ↑ x1+x2 < 3 → z<0 → ŷ<0.5 → 预测 0
```

> **注意**：这组参数是**举例用**（先看图再倒推系数让方程恰好为 \(x_1+x_2=3\)），不是训练得到的最优值；真实场景下由梯度下降学出。同一条边界对应**无数组等价参数**（如 \(w=(2,2),b=-6\)），它们边界相同但 sigmoid 输出的概率陡峭度不同。

## 三 ½、非线性决策边界（特征工程视角）

### 关键洞察：模型还是线性的，只是"特征"变非线性了

逻辑回归的决策边界**对参数 \(\boldsymbol{w}\) 是线性的**——这就是它叫"线性分类器"的原因。但如果在送进模型之前，**先把原始输入做非线性变换**，决策边界在原空间里就能变成曲线。

> ⚡ 核心思想：模型对**参数 \(\boldsymbol{w}\)** 仍然线性（凸优化、有闭式梯度），但对**输入 \(\boldsymbol{x}\)** 已是非线性。

### 例 1：圆形决策边界

构造特征 \(x_1^2,\ x_2^2\)：

\[
z = w_1 x_1^2 + w_2 x_2^2 + b
\]

取 \(w_1 = 1, w_2 = 1, b = -1\)，决策边界 \(z = 0\)：

\[
x_1^2 + x_2^2 = 1 \quad \text{（单位圆）}
\]

圆内 \(z<0\) 预测 0，圆外 \(z>0\) 预测 1。

### 例 2：常见二次边界一览

| \(w_1\) | \(w_2\) | \(b\) | 边界方程 | 形状 |
|---------|---------|-------|---------|------|
| 1 | 1 | -1 | \(x_1^2 + x_2^2 = 1\) | 圆 |
| 1 | 4 | -4 | \(x_1^2 + 4x_2^2 = 4\) | 椭圆 |
| 1 | -1 | -1 | \(x_1^2 - x_2^2 = 1\) | 双曲线 |
| 0 | 1 | -1 | \(x_2^2 = 1\) | 两条平行线 |

### 例 3：高阶多项式

加入 \(x_1, x_2, x_1^2, x_1 x_2, x_2^2, x_1^3, \dots\)：

\[
z = w_1 x_1 + w_2 x_2 + w_3 x_1^2 + w_4 x_1 x_2 + w_5 x_2^2 + \dots + b
\]

阶数越高，可拟合的形状越复杂（理论上可逼近任意边界），但**过拟合风险急剧增加**——必须配 L2 正则。

### 整体工作流

```
原始数据 (x_1, x_2)
        │
        ▼
特征工程：人工/自动构造非线性特征
   φ(x) = [1, x_1, x_2, x_1², x_1 x_2, x_2², ...]
        │
        ▼
逻辑回归（仍是线性模型！只是输入变成了 φ(x)）
   z = w^T φ(x) + b
        │
        ▼
梯度下降学习 w 和 b
        │
        ▼
画出决策边界 z = 0  → 在原始 (x_1, x_2) 空间里看就是曲线
```

### sklearn 标准操作（**记住这段代码**）

```python
from sklearn.preprocessing import PolynomialFeatures, StandardScaler
from sklearn.linear_model import LogisticRegression
from sklearn.pipeline import Pipeline

pipe = Pipeline([
    ("poly",   PolynomialFeatures(degree=2, include_bias=False)),  # ← 关键：升维
    ("scaler", StandardScaler()),                                   # 多项式后必做归一化
    ("clf",    LogisticRegression(C=1.0, penalty="l2", max_iter=1000)),
])
pipe.fit(X, y)
```

`PolynomialFeatures(degree=2)` 自动把 \([x_1, x_2]\) 扩展为 \([x_1, x_2, x_1^2, x_1 x_2, x_2^2]\)。**参数 \(\boldsymbol{w}\) 不是手动设的，是梯度下降学出来的**——你只负责挑特征。

### 几何直觉：升维后变线性可分

```
原空间 (x_1, x_2)：圆形不可分          升维到 (x_1², x_2²)：直线可分

   x_2                                    x_2²
    ✕ ✕ ✕                                   |   ✕ ✕ ✕
  ✕  ○ ○  ✕         ────►                  | ✕         ✕
  ✕  ○ ○  ✕                                |————————————(超平面)
    ✕ ✕ ✕                                   | ○ ○ ○
                                            +—————————————— x_1²
```

> 💡 这正是 **SVM 核技巧（kernel trick）** 的思想——SVM 用核函数**隐式**升维（避开显式构造高维特征的内存开销），而逻辑回归是**显式**升维。两者本质是一回事，只是工程实现路径不同。

### 与神经网络的关系

逻辑回归 + 多项式特征 ≈ **手工版的特征学习**。但人工挑特征痛苦且有限——**神经网络做的事情就是让模型自动学习这些非线性变换**：

```
逻辑回归：       人工构造 φ(x) → 线性 → sigmoid
神经网络：       让网络自己学 φ(x) → 线性 → sigmoid
                 ↑
                 这就是隐藏层在做的事
```

> 一句话：**神经网络 = 自动学习版的"特征工程 + 逻辑回归"**。

### 常见疑问

| 疑问 | 答案 |
|------|------|
| **阶数 degree 怎么选？** | 交叉验证（GridSearchCV），工业界常用 2 或 3，必配 L2 正则 |
| **为什么必须归一化？** | \(x^3\) 比 \(x\) 大几个数量级，不归一化会让梯度爆炸、正则失衡 |
| **特征数爆炸怎么办？** | \(d\) 维做 \(p\) 阶多项式有 \(\binom{d+p}{p}\) 个特征（10 维 3 阶 = 286 个）。用 L1 正则做特征选择，或改用核 SVM / 神经网络 |
| **只能用多项式吗？** | 不！任何非线性变换都可以：交叉 \(x_1 x_2\)（CTR 必备）、分箱 One-Hot、\(\log x\)、\(\sin x\)（周期性）等 |

### 完整可视化代码（圆形数据 → 圆形边界）

```python
import numpy as np, matplotlib.pyplot as plt
from sklearn.datasets import make_circles
from sklearn.preprocessing import PolynomialFeatures, StandardScaler
from sklearn.linear_model import LogisticRegression
from sklearn.pipeline import Pipeline

# 1. 生成同心圆数据（线性不可分）
X, y = make_circles(n_samples=400, factor=0.4, noise=0.1, random_state=0)

# 2. 多项式特征 + 逻辑回归
pipe = Pipeline([
    ("poly",   PolynomialFeatures(degree=2, include_bias=False)),
    ("scaler", StandardScaler()),
    ("clf",    LogisticRegression(C=10, max_iter=1000)),
]).fit(X, y)

# 3. 画决策边界
xx, yy = np.meshgrid(np.linspace(-1.5, 1.5, 300), np.linspace(-1.5, 1.5, 300))
prob = pipe.predict_proba(np.c_[xx.ravel(), yy.ravel()])[:, 1].reshape(xx.shape)

plt.contourf(xx, yy, prob, levels=20, cmap="RdBu_r", alpha=.7)
plt.contour(xx, yy, prob, levels=[0.5], colors="black", linewidths=2)  # 决策边界
plt.scatter(*X.T, c=y, edgecolor="k")
plt.title("Logistic Regression + Polynomial Features → 圆形决策边界")
plt.show()
```

跑一下你会看到：**直线模型在原始 \((x_1, x_2)\) 空间里画出了一个圆**——这就是非线性决策边界的真相。

## 四、损失函数：交叉熵（Cross-Entropy）

### 直觉版：分两种情况看 \(-\log\)（吴恩达课程讲法）

吴恩达课程里先用**分段函数**给出 loss，便于建立直觉：

\[
L\big(f(\boldsymbol{x}), y\big) = \begin{cases}
-\log\big(f(\boldsymbol{x})\big), & \text{if } y = 1 \\
-\log\big(1 - f(\boldsymbol{x})\big), & \text{if } y = 0
\end{cases}
\]

其中 \(f(\boldsymbol{x}) = \sigma(\boldsymbol{w}^\top \boldsymbol{x} + b) \in (0,1)\) 是模型预测"\(y=1\) 的概率"。

#### 关键：\(-\log(x)\) 函数的形状

```
   -log(x)
       |  *
     5 |   *
       |    *
     3 |     *
       |      **
     1 |        ***
     0 +———————————————*— x
       0  0.2  0.5    1
```

| 输入 \(x\) | \(-\log(x)\) | 含义 |
|------|------|------|
| \(x \to 1\) | \(\to 0\) | **不罚** |
| \(x = 0.5\) | \(\approx 0.69\) | 中度惩罚（瞎猜）|
| \(x \to 0\) | \(\to +\infty\) | **无穷大惩罚！** |

> 💡 \(-\log\) 是一个"**离 1 越远，惩罚越狠，且离 0 越近时惩罚增长得超快**"的函数——这正是分类任务想要的。

#### 两个分支的"奖惩表"

**分支 1：\(y=1\) 时 loss = \(-\log(f)\)**

| 真实 \(y\) | 预测 \(f\) | loss | 解读 |
|------|------|------|------|
| 1 | 0.99 | 0.01 | 预测对了且自信 → **几乎不罚** ✅ |
| 1 | 0.5 | 0.69 | 模型瞎猜 → 中度惩罚 |
| 1 | 0.01 | 4.60 | 预测反了且自信 → **重罚！** ❌ |
| 1 | → 0 | → \(+\infty\) | 完全错误 💀 |

**分支 2：\(y=0\) 时 loss = \(-\log(1-f)\)**（图形是分支 1 的水平镜像）

| 真实 \(y\) | 预测 \(f\) | \(1-f\) | loss | 解读 |
|------|------|------|------|------|
| 0 | 0.01 | 0.99 | 0.01 | 预测对了 → **几乎不罚** ✅ |
| 0 | 0.5 | 0.5 | 0.69 | 瞎猜 |
| 0 | 0.99 | 0.01 | 4.60 | 预测反了 → **重罚！** ❌ |

> 注意分支 2：log 里面是 \(1 - f\)，因为它代表"模型认为是负类的概率"。当真实是 0 时，希望 \(1-f\) 接近 1。

#### 用 \(y\) 当"开关"合并成一个公式

分段函数可以用一个聪明的小技巧合并：

\[
\boxed{
L\big(f(\boldsymbol{x}), y\big) = -y \log f(\boldsymbol{x}) - (1 - y) \log\big(1 - f(\boldsymbol{x})\big)
}
\]

**验证**：
- \(y = 1\)：\(-1 \cdot \log f - 0 \cdot \log(1-f) = -\log f\) ✅
- \(y = 0\)：\(-0 \cdot \log f - 1 \cdot \log(1-f) = -\log(1-f)\) ✅

\(y\) 和 \(1-y\) 像两个开关，正好关掉一项、保留一项——**这就是分段写法的统一版本**。

整体 cost：
\[
J(\boldsymbol{w}, b) = \frac{1}{m} \sum_{i=1}^{m} L\big(f(\boldsymbol{x}^{(i)}), y^{(i)}\big)
\]

> 📌 关于 \(\frac{1}{2}\) 的位置：吴恩达在线性回归里把 \(\frac{1}{2}\) 放进 loss 是为了让 cost 公式 \(\frac{1}{m}\sum L^{(i)}\) 在所有模型间保持统一形式。**但逻辑回归的 loss 里没有 \(\frac{1}{2}\)**——交叉熵求导本身就很干净（\(\partial L / \partial z = \hat{y} - y\)），不需要这个常数。

---

### 推导：从 MLE 出发（数学版）

直觉解释完了，现在看为什么数学上"必然"是 \(-\log\)——这不是凑巧，是**最大似然估计**的自然结果。

设训练集 \(\{(\boldsymbol{x}_i, y_i)\}_{i=1}^n\)，样本独立。似然函数：
\[
L(\boldsymbol{w}) = \prod_{i=1}^{n} \hat{y}_i^{y_i} (1-\hat{y}_i)^{1-y_i}
\]

> 验证伯努利写法：\(y_i = 1\) 时为 \(\hat{y}_i\)；\(y_i = 0\) 时为 \(1 - \hat{y}_i\)——刚好对应"模型预测真实标签的概率"。

最大化似然 = 让训练数据"出现得最自然"。但**连乘很难算**（数值会下溢）：
\[
0.9 \times 0.8 \times \dots \times 0.6 \to \text{超小的数}
\]

**取 log 把连乘变连加**（log 单调，不改变最优解位置）：
\[
\log L(\boldsymbol{w}) = \sum_{i=1}^{n}\Big[ y_i \log \hat{y}_i + (1-y_i) \log(1-\hat{y}_i) \Big]
\]

加个负号把"最大化"变"最小化"（习惯）：

\[
\boxed{
J(\boldsymbol{w}) = -\frac{1}{n}\sum_{i=1}^{n}\Big[ y_i \log \hat{y}_i + (1-y_i) \log(1 - \hat{y}_i) \Big]
}
\]

这就是**二元交叉熵（Binary Cross-Entropy, BCE）**。

> ⚡ **大彻大悟**：log 不是"为了惩罚越来越大"才人为引入的，它是 **最大似然 + 连乘变连加** 的数学产物。"惩罚直觉"和"数学推导"在这里完美契合——这就是为什么 BCE 如此优雅。

### 为什么不用 MSE？（面试高频）

#### 视角 1：凸性
- **MSE + Sigmoid \(\Rightarrow\) 非凸**，存在多个局部极小值，梯度下降可能卡住。
- **BCE + Sigmoid \(\Rightarrow\) 凸函数**，全局最优唯一。

> 直观理解：\(\sigma\) 是非线性的，把 \((y - \sigma(z))^2\) 画出来在 \(z\) 上是 S 曲线的平方，会出现"凸起+平台+凸起"，非凸。

#### 视角 2：梯度消失
- MSE 对 \(z\) 的梯度：\(\frac{\partial L}{\partial z} = (\hat{y} - y) \cdot \sigma'(z) = (\hat{y} - y) \hat{y}(1 - \hat{y})\)
  - 当 \(\hat{y}\) 接近 0 或 1 时，\(\sigma'\) 趋近 0，**梯度饱和、训练停滞**。
- BCE 对 \(z\) 的梯度：\(\frac{\partial L}{\partial z} = \hat{y} - y\)
  - **形式极简**，且 \(\sigma'(z)\) 在分子分母**消掉了**——这就是 BCE + Sigmoid 的"黄金搭档"。

#### 视角 3：MLE 推导自然结果
高斯噪声 \(\Rightarrow\) MSE；伯努利噪声 \(\Rightarrow\) 交叉熵。**任务匹配损失。**

### 视角 4：信息论
\(H(p, q) = -\sum p \log q\)：交叉熵等价于"用预测分布 \(q\) 编码真实分布 \(p\) 所需的平均比特数"，越小说明越接近真实分布。

## 五、参数求解：梯度下降

### 梯度推导（必背）

记 \(z_i = \boldsymbol{w}^\top \boldsymbol{x}_i + b\)，\(\hat{y}_i = \sigma(z_i)\)。链式法则：

\[
\frac{\partial J}{\partial w_j} = \frac{1}{n}\sum_{i=1}^{n} (\hat{y}_i - y_i) x_{ij}
\]

写成矩阵：
\[
\boxed{\nabla_{\boldsymbol{w}} J = \frac{1}{n} X^\top (\hat{\boldsymbol{y}} - \boldsymbol{y})}
\]

> ⚡ **惊人发现**：这个梯度形式和**线性回归的梯度完全一致**！区别只在于 \(\hat{y}\) 的计算（多套了一层 \(\sigma\)）。这就是把它们都归到 GLM 的原因。

### 更新规则

\[
\boldsymbol{w} \leftarrow \boldsymbol{w} - \eta \cdot \nabla_{\boldsymbol{w}} J
\]

| 优化器 | 适用 |
|--------|------|
| **Batch GD** | 数据小、特征少 |
| **SGD** | 在线学习、超大规模数据 |
| **Mini-batch + Adam** | 实战首选 |
| **L-BFGS** | sklearn 默认（数据中等、特征 < 数千） |
| **liblinear** | sklearn 处理小数据 + L1 / L2 |
| **saga** | 大数据 + L1/L2/Elastic Net |

### 没有闭式解

\(J(\boldsymbol{w})\) 中含 \(\log \sigma(\cdot)\)，无法像线性回归那样写出 \((X^\top X)^{-1} X^\top \boldsymbol{y}\) 的解析解，只能数值优化。

## 六、正则化

完全照搬线性回归三件套，只是损失主项换成 BCE：

| 类型 | 公式 | 效果 |
|------|------|------|
| **L2（默认）** | \(J + \lambda \|\boldsymbol{w}\|_2^2\) | 权重压小，缓解过拟合 |
| **L1** | \(J + \lambda \|\boldsymbol{w}\|_1\) | 稀疏，做特征选择 |
| **Elastic Net** | \(J + \lambda_1 \|\boldsymbol{w}\|_1 + \lambda_2 \|\boldsymbol{w}\|_2^2\) | 兼顾稀疏 + 稳定 |

> sklearn 用参数 `C = 1/λ` —— **C 越小正则越强**（反直觉，注意）。

---

## 七、多分类扩展

### 1. Softmax 回归（多项逻辑回归）

把 Sigmoid 推广到 \(K\) 个类别：
\[
P(y=k \mid \boldsymbol{x}) = \frac{e^{\boldsymbol{w}_k^\top \boldsymbol{x}}}{\sum_{j=1}^{K} e^{\boldsymbol{w}_j^\top \boldsymbol{x}}}
\]

损失：**类别交叉熵（Categorical Cross-Entropy）**
\[
J = -\frac{1}{n} \sum_{i=1}^{n} \sum_{k=1}^{K} \mathbb{1}[y_i=k] \log P(y_i=k \mid \boldsymbol{x}_i)
\]

> 当 \(K=2\) 时 Softmax 回归与逻辑回归**等价**（数学上可通过参数减一推出）。

### 2. One-vs-Rest（OvR）/ One-vs-One（OvO）

- **OvR**：训练 \(K\) 个二分类器，每个区分"第 \(k\) 类 vs 其他"。简单、可并行。
- **OvO**：训练 \(\binom{K}{2}\) 个二分类器两两 PK。更准但贵。

`sklearn.LogisticRegression(multi_class='ovr' / 'multinomial')` 切换两种策略。**默认 multinomial（即 Softmax 回归）**。

---

## 八、对数几率（Log-Odds）视角 —— 名字由来

把 \(P(y=1|\boldsymbol{x}) / P(y=0|\boldsymbol{x})\) 称为**几率（odds）**，再取 log：

\[
\log \frac{P(y=1\mid\boldsymbol{x})}{P(y=0\mid\boldsymbol{x})} = \boldsymbol{w}^\top \boldsymbol{x} + b
\]

> **逻辑回归 = 用线性模型拟合"对数几率"**。这就是它叫 *Logistic Regression* 而不是 *Logistic Classification* 的原因——它确实在做"回归"，只不过回归的目标是**对数几率**。

### 权重的可解释性（金融/医学最爱）

\(w_j\) 的含义：**特征 \(x_j\) 每增加 1 单位，对数几率增加 \(w_j\)**，即**几率变成原来的 \(e^{w_j}\) 倍**。

例：风控模型中"逾期次数"的 \(w_j = 0.4\)，则每多 1 次逾期，违约几率乘以 \(e^{0.4} \approx 1.49\)，即提高 49%。

### 广义线性模型（GLM）家族

| 模型 | 分布 | 链接函数 \(g(\mu)\) | 任务 |
|------|------|-------------------|------|
| 线性回归 | 高斯 | 恒等 \(\mu\) | 连续回归 |
| **逻辑回归** | **伯努利** | **logit** \(\log\frac{\mu}{1-\mu}\) | 二分类 |
| Softmax 回归 | 多项 | softmax 链接 | 多分类 |
| 泊松回归 | 泊松 | \(\log \mu\) | 计数数据 |

> 这套统一框架告诉你："线性回归、逻辑回归不是两件事，是同一件事的两种表现。"

---

## 九、优缺点

### 优点
- **训练快、推理极快**（一次内积），CTR 这类"几亿样本 / 十亿特征"场景的工业级首选
- **可解释性强**——权重直接对应特征重要性 + 几率倍数
- **概率输出**——可与业务阈值组合（风控、医学诊断）
- **凸优化**——必收敛到全局最优
- **可在线更新**（FTRL 优化器是工业界标配）

### 缺点
- 决策边界是**线性的**，无法直接处理非线性可分问题（除非配交叉特征 / 核技巧 / 神经网络）
- 对**多重共线性**敏感，会让权重不稳定（用 Ridge 缓解）
- 对**异常值**有一定敏感性（不如 SVM 鲁棒）
- 类别极度不平衡时 0.5 阈值不合理，需重设阈值或改用 PR-AUC

## 十、实战与常见坑

### 1. 必须做的预处理
- **特征归一化**：梯度下降必须；L1/L2 正则系数才公平作用
- **类别变量**：One-Hot（低基数）/ Target Encoding（高基数）
- **缺失值**：填充或加 missing 标记列

### 2. 类别不平衡
| 方法 | 说明 |
|------|------|
| `class_weight='balanced'` | sklearn 自动按频次倒数加权 |
| 过采样（SMOTE） | 合成少数类 |
| 欠采样 | 丢弃多数类样本 |
| 改阈值 | 用 PR 曲线找最佳阈值，而非用 0.5 |
| 改指标 | 看 **PR-AUC**、F1、KS，而非 Accuracy |

### 3. 特征工程比模型更重要（对 LR 尤其）
- LR 是线性模型，**强烈依赖人工交叉特征**（如 `gender × age_bucket`）
- 工业界 CTR 模型：LR + 海量人工交叉特征 → FM → DeepFM 的进化路径

### 4. 概率校准（Calibration）
LR 输出的概率通常**比较准**（自然校准），但加了正则或重采样后会偏。可用：
- `sklearn.calibration.CalibratedClassifierCV`
- 可靠性曲线（reliability diagram）诊断

### 5. 数值稳定
\(\sigma(z)\) 当 \(z\) 很大很小时溢出。实现时用：
\[
\log(1 + e^{z}) = \max(z, 0) + \log(1 + e^{-|z|})
\]
（`numpy.logaddexp` / PyTorch `BCEWithLogitsLoss` 都内置了这个技巧。）

## 十一、手写代码（面试级）

### NumPy 从零实现

```python
import numpy as np

def sigmoid(z):
    # 数值稳定版本，避免 exp 溢出
    return np.where(z >= 0,
                    1.0 / (1.0 + np.exp(-z)),
                    np.exp(z) / (1.0 + np.exp(z)))

class LogisticRegressionScratch:
    """支持 L2 正则的二分类逻辑回归（mini-batch GD）"""

    def __init__(self, lr=0.1, epochs=1000, l2=0.0, batch_size=None, verbose=False):
        self.lr, self.epochs, self.l2 = lr, epochs, l2
        self.batch_size, self.verbose = batch_size, verbose
        self.history = []

    def _add_bias(self, X):
        return np.c_[np.ones(len(X)), X]

    def fit(self, X, y):
        X = self._add_bias(X)
        n, d = X.shape
        self.w = np.zeros(d)
        bs = self.batch_size or n

        for epoch in range(self.epochs):
            idx = np.random.permutation(n)
            for start in range(0, n, bs):
                b = idx[start:start + bs]
                Xb, yb = X[b], y[b]
                y_hat = sigmoid(Xb @ self.w)
                grad = Xb.T @ (y_hat - yb) / len(yb)
                # L2 正则不罚偏置
                grad[1:] += self.l2 * self.w[1:]
                self.w -= self.lr * grad

            # 记录整个数据集的 loss
            y_hat_full = sigmoid(X @ self.w)
            eps = 1e-12
            loss = -np.mean(y * np.log(y_hat_full + eps) +
                            (1 - y) * np.log(1 - y_hat_full + eps))
            self.history.append(loss)
            if self.verbose and epoch % 100 == 0:
                print(f"epoch {epoch:4d}  loss={loss:.4f}")
        return self

    def predict_proba(self, X):
        return sigmoid(self._add_bias(X) @ self.w)

    def predict(self, X, threshold=0.5):
        return (self.predict_proba(X) >= threshold).astype(int)
```

### sklearn 标准用法

```python
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.linear_model import LogisticRegression
from sklearn.model_selection import cross_val_score

pipe = Pipeline([
    ("scaler", StandardScaler()),
    ("clf", LogisticRegression(
        penalty="l2",       # "l1" / "l2" / "elasticnet" / "none"
        C=1.0,              # 正则强度倒数：C 越小正则越强
        solver="lbfgs",     # liblinear/saga/newton-cg/...
        class_weight="balanced",
        max_iter=1000,
    )),
])

# 用 ROC-AUC 作为评估（类别不均衡时优于 accuracy）
scores = cross_val_score(pipe, X, y, cv=5, scoring="roc_auc")
print(f"ROC-AUC: {scores.mean():.4f} ± {scores.std():.4f}")
```

### 决策边界可视化（2D）

```python
import numpy as np, matplotlib.pyplot as plt
from sklearn.datasets import make_classification
from sklearn.linear_model import LogisticRegression

X, y = make_classification(n_features=2, n_redundant=0, n_clusters_per_class=1,

xx, yy = np.meshgrid(np.linspace(*X[:,0].min() - 1, X[:,0].max() + 1, 200),

plt.title("Logistic Regression — decision boundary is linear")
```

## 十三、与其他模型的关系图

```
                                   线性回归
                                       │ (高斯, 恒等链接)
                                       ▼
                            广义线性模型 (GLM)
                          ┌────────────┼─────────────┐
                          ▼            ▼             ▼
                  逻辑回归        Softmax 回归     泊松回归
                (伯努利, logit)  (多项, softmax)  (泊松, log)
                          │
                  深度化 │ 加隐藏层
                          ▼
                  神经网络（最后一层 = 逻辑回归 / Softmax）
```

> **一句话总结**：逻辑回归 = 单层神经网络（无隐藏层）+ Sigmoid 激活 + BCE 损失。理解了它，就理解了一半的深度学习。
