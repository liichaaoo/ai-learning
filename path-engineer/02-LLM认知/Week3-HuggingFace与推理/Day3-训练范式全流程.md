## 0. 心法（5 分钟）

> **从零养一个 ChatGPT 三步走：**
>
> ```
> ① Pre-training（预训练）：让模型"读书"——见识互联网级别的文本
> ② SFT（监督微调）：让模型"懂礼貌"——按 instruction 回答
> ③ RLHF / DPO（对齐）：让模型"懂喜好"——回答得更符合人类偏好
> ```

每一步都把上一步的模型当起点继续训。

---

## 1. 一图速记：三阶段全流程 ⭐（10 分钟）

```
┌──────────────────────────────────────────────────────────────────────┐
│                                                                      │
│  ① Pre-training（预训练）                                           │
│     输入：互联网海量文本（书籍 + 网页 + 代码）→ TB 级数据           │
│     训练：预测下一个 token（Next Token Prediction）                 │
│     代价：💰💰💰 上千万美元 / 几千张 H100 / 几个月                  │
│     输出：Base Model（如 Llama-3-Base，会续写但不会"对话"）        │
│                              │                                       │
│                              ↓                                       │
│  ② SFT（Supervised Fine-Tuning，监督微调）                          │
│     输入：高质量 (instruction, response) 对，~ 数万条                │
│     训练：让模型学会"按 instruction 回答"                           │
│     代价：💰 几万美元 / 几张 GPU / 几天                             │
│     输出：SFT Model（已经能对话，但回答可能不够好）                 │
│                              │                                       │
│                              ↓                                       │
│  ③ Alignment（对齐）                                                │
│     方法 A：RLHF（人类反馈强化学习）                                │
│       1) 训练 Reward Model（给回答打分）                            │
│       2) 用 PPO 等强化学习算法优化模型                              │
│     方法 B：DPO（Direct Preference Optimization，2023+）⭐          │
│       直接用偏好数据微调，不用 Reward Model                         │
│     输出：Chat Model（如 Llama-3-Instruct，能"对齐人类偏好"）       │
│                                                                      │
└──────────────────────────────────────────────────────────────────────┘
```

> 💡 **能背出这张图 + 3 阶段的输入输出，今天就通关了。**

## 2. Pre-training：让模型"读书"（15 分钟）

### 2.1 训练目标：预测下一个 token

```
"今天天气真"  →  模型预测下一个 token：
                    "好" (0.4)
                    "热" (0.2)
                    "冷" (0.15)
                    ...

如果真值是 "好"，loss = -log(0.4) ≈ 0.92
反向传播 → 更新参数
```

> 💡 **回到 Week 2 Day 5**：这就是 GPT 的 CLM（Causal Language Model）训练目标，整本互联网的文本都被这样过一遍。

### 2.2 数据规模（开眼界）

| 模型 | 训练数据 | 参数量 |
|------|---------|--------|
| GPT-3 (2020) | 300B tokens | 175B |
| Llama 1 (2023) | 1.4T tokens | 65B |
| Llama 2 (2023) | 2T tokens | 70B |
| **Llama 3 (2024)** | **15T tokens** | 70B |
| **Qwen 2.5 (2024)** | **18T tokens** | 72B |

> 🤯 **15T tokens** ≈ 100 亿本书 ≈ **整个互联网公开文本的相当一部分**。

### 2.3 算力代价

```
Llama 3 70B 训练用了约：
  - 16,000 张 H100 GPU
  - 训练 ~ 54 天
  - 估算成本：~ $50M ~ $100M（数千万美元）
```

> 💡 **作为工程师你需要知道**：**预训练你做不了**（除非你在大厂训练团队）。**SFT/LoRA 才是工程师的赛道**。

### 2.4 Pre-training 之后的模型能干什么

**Base 模型（如 Llama-3-Base）**：
- ✅ 会续写文本（"今天天气真好" → "，适合出去散步……"）
- ❌ **不会对话**（你问它"什么是机器学习"，它可能回答"是个好问题，请再思考一下……"——它只是在续写常见后续）

> 🎯 **Base 模型 ≠ 对话模型**——这是 SFT 要解决的。

## 3. SFT：让模型"懂礼貌"（15 分钟）

### 3.1 SFT 在干什么

**用人工标注的 (instruction, response) 对训练**，让模型学会按指令回答。

数据样例：

```json
{
  "instruction": "用一句话解释什么是机器学习",
  "response": "机器学习是让计算机从数据中自动学习规律的技术。"
}
{
  "instruction": "把这段中文翻译成英文：今天天气很好",
  "response": "The weather is nice today."
}
```

### 3.2 训练时的"作弊"细节

```
prompt: "用一句话解释什么是机器学习\n答："
target: "机器学习是让计算机从数据中自动学习规律的技术。"

训练时，模型只对 target 部分计算 loss——
prompt 部分不参与训练（mask 掉）。

这样模型才学会"看到 prompt 后该输出什么"，而不是学怎么续写 prompt。
```

### 3.3 数据规模（对比 Pre-training）

| 阶段 | 数据量 | 数据来源 |
|------|--------|---------|
| Pre-training | TB 级 / 万亿 token | 全网爬取 |
| **SFT** | **万到几十万条** | **人工编写或半自动生成** |

> 💡 **数据质量 > 数量**——10 万条精心标注的数据，比 1000 万条机器生成的有效得多。
>
> 这就是为什么 OpenAI 当年请了大量 PhD 写示例数据。

### 3.4 SFT 之后的模型

**SFT Model（如 Llama-3-SFT，闭源）**：
- ✅ 能对话、按指令回答
- ❌ 但回答可能"不够人类"——比如一些回答虽然正确，但语气生硬、不够有用

> 🎯 **SFT 之后的模型 ≈ 一个能干但不会察言观色的实习生**。这就是 RLHF 要解决的。

## 4. RLHF：让模型"懂人类喜好"（20 分钟）

### 4.1 为什么需要 RLHF

**SFT 的局限**：
- 标注数据再多，也覆盖不了所有可能的提问
- 同一个问题，"正确答案"可能有很多种——哪种更**好**？

**RLHF 的思路**：让人类给回答打分，**学一个"打分器"**，再用强化学习优化模型。

### 4.2 RLHF 三步走

```
┌─ Step 1：收集人类偏好数据 ─────────────────────────────────┐
│  对每个 prompt 让模型生成 2-4 个回答                       │
│  让人类标注："A 比 B 好" / "C 比 D 好"                     │
│  得到偏好对 (chosen, rejected)                             │
└────────────────────────────────────────────────────────────┘
                            ↓
┌─ Step 2：训练 Reward Model（奖励模型）────────────────────┐
│  把上面收集的偏好对喂给一个小模型                           │
│  让它学会：给"好回答"打高分，给"差回答"打低分             │
│  输出：一个能给任意 (prompt, response) 打分的模型          │
└────────────────────────────────────────────────────────────┘
                            ↓
┌─ Step 3：用 PPO 强化学习优化 LLM ──────────────────────────┐
│  让 LLM 生成回答 → Reward Model 打分 → 高分激励 LLM        │
│  反复迭代，让 LLM 输出越来越得高分（= 越来越被人喜欢）     │
│                                                            │
│  PPO（Proximal Policy Optimization）= 强化学习的一种算法   │
└────────────────────────────────────────────────────────────┘
```

### 4.3 RLHF 的工程难度

```
   Pre-training  ⭐⭐⭐⭐⭐（贵但流程标准）
   SFT          ⭐⭐         （简单，纯监督学习）
   RLHF         ⭐⭐⭐⭐     （难！强化学习超不稳定）
```

> 💡 **RLHF 训练极其难调**——这是 OpenAI/Anthropic 等头部团队真正的护城河。

### 4.4 DPO：RLHF 的"偷懒版"⭐（2023 后流行）

**DPO（Direct Preference Optimization）**：

```
RLHF：偏好对 → 训 Reward Model → 用 RL 优化 LLM
DPO：偏好对 → 直接构造一个 loss → 直接微调 LLM   ← 跳过 Reward Model
```

**好处**：
- 不用训 Reward Model
- 不用 PPO（强化学习），用普通监督学习就能搞
- **效果接近 RLHF，但训练稳定 10 倍**

> 🎯 **现在很多开源对齐方案都用 DPO**（包括 Llama 3 / Qwen）——它就是 RLHF 的"工程实用版"。
>
> **听过 DPO 这个词，比能推导它公式更重要**。

## 5. 还有哪些"训练范式"（5 分钟，听过即可）

| 范式 | 简介 | 用在哪 |
|------|------|------|
| **CPT**（Continued Pre-Training）| 在垂直领域语料上继续预训练 | 法律 / 医疗 / 金融大模型 |
| **Instruction Tuning** | 大量 instruction 数据 SFT | 通用对话能力 |
| **Tool Use Training** | 教模型"调用工具" | Agent / Function Calling |
| **RLAIF**（RL from AI Feedback）| 用 AI（强模型）打分代替人 | 降低标注成本 |
| **Constitutional AI** | Anthropic 的 RLAIF 变种 | Claude 系列 |

> 💡 **本周认知层够用**——阶段 5 微调时再展开 SFT / DPO 实操。

                          │ 用预训练好的开源模型
                          ↓

                          │
                          ↓
                 ┌── DPO/RLHF ───┐
                 │  ⚠️ 慎用      │  → 大公司或专门团队
                 └────────────────┘
```

### 工程师 80% 时间在做的事

| 阶段 | 做什么 | 难度 |
|------|--------|------|
| **选模型** | 看场景选 Qwen / Llama / GPT-4o | ⭐ |
| **写 Prompt** | Prompt Engineering | ⭐⭐ |
| **RAG** | 检索增强生成（阶段 3）| ⭐⭐⭐ |
| **微调（LoRA）** | 垂直领域 SFT（阶段 5）| ⭐⭐⭐ |
| **部署 / 量化** | 把模型上生产（阶段 6）| ⭐⭐⭐ |

> 🎯 **作为后端工程师**：**80% 用现成模型 + Prompt + RAG 解决问题，20% 微调。** Pre-training / RLHF 几乎不会碰。
