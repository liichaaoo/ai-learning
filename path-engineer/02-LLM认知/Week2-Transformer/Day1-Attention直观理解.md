## 0. 心法（5 分钟）

> **Attention 的本质：让模型在处理每个词时，都能"挑出"上下文里最相关的几个词来参考。**

举个最直白的例子：

```
"小明把书放在桌上，他离开时忘了拿它。"

  当模型看到 "他" 时 → 应该重点看回 "小明"
  当模型看到 "它" 时 → 应该重点看回 "书"
```

**Attention 解决的核心问题**：让"他/它"知道自己应该往哪里"看回去"。

> 🧠 **Java 类比**：把 Attention 想成一个 `Map<Token, Weight>`——对句子里每个 token 都生成一份"我跟谁最相关"的权重表，然后按权重加权求和上下文。

---

## 1. 为什么需要 Attention：RNN 时代的痛（15 分钟）

### 1.1 在 Transformer 之前，处理序列靠 RNN

RNN（包括 LSTM/GRU）的工作方式：

```
词1 → [神经元状态 h1] → 词2 → [h2] → 词3 → [h3] → ... → 词N → [hN]
                                                                  ↓
                                                                 输出
```

**特点**：从左到右**串行**处理，把"已经看过的内容"压缩进一个隐状态向量 `h` 里。

### 1.2 RNN 的三大痛点

| 痛点 | 现象 | 类比 |
|------|------|------|
| **长距离依赖丢失** | 第 100 个词很难记住第 1 个词 | 你转述一段话，越往后越记不清开头 |
| **无法并行** | 必须先算完 h1 才能算 h2 | 流水线工人一个挨一个，没法多人同时干 |
| **信息瓶颈** | 整段历史都压进一个固定大小的 `h` 向量 | 用一个便利贴写完整本书的剧情 |

> 💡 **关键认知**：RNN 之所以在长文本上吃力，根本原因是**信息流是「单向、串行、强压缩」的**。

### 1.3 Attention 怎么解决这三个问题

| RNN 痛点 | Attention 怎么破 |
|---------|----------------|
| 长距离依赖丢失 | **任意两个位置直接连一根线**，距离再远也是一步到位 |
| 无法并行 | **所有位置一次性算完**——矩阵乘法天然并行 |
| 信息瓶颈 | **每个词都保留自己的向量**，不再压成一个 h |

> 🎯 **一句话总结**：**Attention = 把"串行压缩"换成"全连接的加权选择"。**

## 2. "看上下文"到底是什么意思（15 分钟）

### 2.1 一个具体例子

句子：`The animal didn't cross the street because it was too tired.`

问题：`it` 指的是什么？是 `animal` 还是 `street`？

人类一眼能看出是 `animal`（因为 `tired`——街道不会累）。但模型怎么知道？

### 2.2 Attention 给出的答案

模型在处理 `it` 这个词时，会**对句子里的每个词都打一个相关性分数**：

```
            处理 "it" 这个位置时，看其他位置的注意力权重：

The       ▮▮          (0.05)
animal    ▮▮▮▮▮▮▮▮▮  (0.45) ← 最高！
didn't    ▮            (0.02)
cross     ▮▮          (0.05)
the       ▮            (0.02)
street    ▮▮▮▮        (0.18)
because   ▮▮          (0.05)
it        ▮▮          (0.06)
was       ▮▮          (0.05)
too       ▮▮          (0.04)
tired     ▮▮▮         (0.13)
```

然后用这些权重去**加权求和**所有词的向量，得到 `it` 这个位置的"上下文增强表示"——这个新向量就是"这个 it 是 animal 的代词"的语义。

> 💡 **核心动作**：每个位置都干两件事
> 1. **打分**：跟其他位置的相关性
> 2. **加权求和**：按相关性混合上下文信息

## 3. 图书馆类比：为什么有 Q / K / V 三件套（20 分钟）

> 这是 Day 2 公式的灵魂入口，**今天先把直觉建好，明天看公式会非常顺**。

### 3.1 场景：你去图书馆查资料

假设你去图书馆找"机器学习"相关的书，整个过程：

```
┌─────────────────────────────────────────────────────────────┐
│  ① 你心里有个查询词："机器学习"            ← Query (Q)     │
│                                                             │
│  ② 图书馆每本书都有一个标签："数学"、       ← Key   (K)     │
│      "AI"、"小说"、"诗歌" ……                                │
│                                                             │
│  ③ 每本书的实际内容、知识                   ← Value (V)     │
│                                                             │
│  ④ 你拿 Query 跟所有 Key 比，找最匹配的那几本              │
│  ⑤ 然后把这几本的 Value 内容综合起来读                     │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 把这个类比映射回 Attention

| 图书馆 | Self-Attention |
|------|---------------|
| 你心里的查询词 "机器学习" | **Query（Q）**：当前位置想"问"什么 |
| 每本书的标签 | **Key（K）**：每个位置"对外的标签" |
| 每本书的内容 | **Value（V）**：每个位置真正携带的信息 |
| Query 和 Key 算相似度 | **`Q · K^T`** 点积，得相似度分数 |
| 加权读取多本书的内容 | **`softmax(...) · V`** 加权求和 |

### 3.3 关键反直觉：**Q、K、V 都来自同一句话**

在 **Self-**Attention 里，对于句子 `[小明, 把, 书, 放, 在, 桌上]`：

- 每个词都同时**生成自己的 Q、K、V 三个向量**
- 词 i 用自己的 Q 跟所有词（包括自己）的 K 比，得到一组权重
- 用这组权重，去加权求和所有词的 V

> 💡 **"Self" 的含义**：自己跟自己（同一个序列）做 Attention。
>
> 在 RAG 里你会看到的 "Cross-Attention" 才是 "查询和文档"分别提供 Q 和 K/V 的——本周不展开。

### 3.4 那 Q / K / V 的向量是怎么来的？

非常简单：**对每个词的输入向量 x，分别乘上三个矩阵 W_Q / W_K / W_V**：

```
对每个词 x_i：
    Q_i = x_i · W_Q
    K_i = x_i · W_K
    V_i = x_i · W_V
```

`W_Q`、`W_K`、`W_V` 是**模型要学习的参数**——训练完成后，模型自己学会了"该怎么投影一个词，让它的 Q 能问出有用的问题、K 能挂出准确的标签、V 能携带合适的信息"。

> 🎯 **本节最重要的一句话**：Q / K / V 是同一个 x **被三个不同矩阵投影出来的三种"角色"**——一个负责问、一个负责挂标签、一个负责传信息。

## 4. 一段最简代码：用 NumPy 实现 Attention 直觉（25 分钟）

> 这段代码不需要训练，**只是为了让你"看见 Attention 在干什么"**——形状变化、softmax 后的权重长什么样。

```python
import numpy as np

np.random.seed(42)

# ─── 假设我们有一个长度 4 的序列，每个 token 维度 8 ───
seq_len, d = 4, 8
x = np.random.randn(seq_len, d)        # (4, 8)：4 个 token，每个 8 维

# ─── 三个学习矩阵（这里直接随机初始化，相当于"未训练状态"）───
W_Q = np.random.randn(d, d) * 0.1
W_K = np.random.randn(d, d) * 0.1
W_V = np.random.randn(d, d) * 0.1

# ─── 第 1 步：把 x 投影成 Q、K、V ───
Q = x @ W_Q                            # (4, 8)
K = x @ W_K                            # (4, 8)
V = x @ W_V                            # (4, 8)

# ─── 第 2 步：算相似度矩阵（每个 token 跟每个 token 的关系）───
scores = Q @ K.T                       # (4, 4)：第 i 行第 j 列 = token i 看 token j 的分数

# ─── 第 3 步：softmax（按行归一化成"权重"）───
def softmax(x, axis=-1):
    e = np.exp(x - x.max(axis=axis, keepdims=True))
    return e / e.sum(axis=axis, keepdims=True)

attn_weights = softmax(scores, axis=-1)   # (4, 4)：每行加起来 = 1

# ─── 第 4 步：加权求和 V ───
output = attn_weights @ V              # (4, 8)：每个 token 的"上下文增强表示"

print("注意力权重（每行加起来=1）：")
print(np.round(attn_weights, 3))
print("\n输出形状：", output.shape)     # (4, 8) —— 跟输入一样！
```

### 看这段代码要带走的三件事

1. **形状全程不变**：进来 `(4, 8)`，出去也是 `(4, 8)`——所以 Transformer 可以堆 N 层
2. **`attn_weights` 每行加起来 = 1**：这是 softmax 的功劳，权重的总量被归一化
3. **整个过程只用了 3 个矩阵乘 + 1 个 softmax**：没有循环、可以 GPU 并行

> 🎯 **如果你看完这段代码能讲清楚每一步的形状和含义**，今天的目标就达成了。

## 5. 今天的"那一刻顿悟"（5 分钟）

回到开头那句话：

> **Attention 的本质：让模型在处理每个词时，都能"挑出"上下文里最相关的几个词来参考。**

现在你应该理解：

```
"挑出"      ← 用 Q 和 K 算分数 + softmax 归一化
"几个词"    ← 权重高的那几个位置
"参考"      ← 用权重加权求和 V
```

而 Q / K / V 三件套，本质上是**让模型自己学会"该怎么查、该挂什么标签、该携带什么信息"**——而不是写死规则。

> 💡 **明天 Day 2 我们就把这一切翻成一行公式**：`Attention(Q, K, V) = softmax(QK^T / √d_k) V`
>
> 你会发现：今天讲的每一个步骤，都对应公式里的一小段。
