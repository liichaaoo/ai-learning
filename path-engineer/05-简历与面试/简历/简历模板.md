# 简历模板（Java 后端 → AI 应用工程师）

> 📅 **整理时间**：2026.05
> 🎯 **适用对象**：5+ 年 Java 后端，转型 Spring AI / RAG / Agent 应用方向
> 📌 **使用方式**：先读 [`./对标简历参考.md`](./对标简历参考.md)，再用本模板填充
> 📏 **长度控制**：**1.5 页**（资深 = 1 页太短，2 页 HR 嫌长，1.5 页最佳）
> 🎨 **排版工具**：推荐用 [超级简历](https://www.wondercv.com/) 或 [WonderCV] 直接套模板

---


## 一、个人信息

```
张某某 ▎8 年 Java 后端 + 1 年 AI 应用 ▎P6/T5 ▎期望：AI 应用工程师 / 高级后端
📞 138-xxxx-xxxx  ✉️ xxx@gmail.com  🔗 github.com/xxx  📍 深圳，可远程
```

**4 个关键设计**：

1. **姓名后跟"年限+方向"**：HR 0.5 秒判断你属于哪条线
2. **职级写实际等级**：腾讯 P6 / 阿里 P7 / 字节 2-1，让对方对得上
3. **期望岗位明确**：不要写"高级开发"这种含糊词，要写具体方向
4. **联系方式紧凑**：一行容纳全部，节省第一屏空间

## 二、个人摘要（Summary）—— **决定 HR 是否往下看**

### 📌 模板（做完阶段 3 后可用）

```
8 年 Java 后端开发经验，深度参与企业级中间件和 AI 应用平台建设。
精通 Spring 生态（Boot 3 / Cloud / Security），近一年专注 Spring AI 与 RAG 架构，
主导企业 RAG 知识库从 0 到 1 落地，覆盖 200+ 内部文档 / 50+ QPS。
擅长高并发架构设计与性能调优（千万级 DAU / QPS 万级 经验），
具备从需求分析、架构设计、技术选型到上线运维的完整闭环能力。
```

### 🎯 5 句话的设计意图

| 句序 | 作用 | 关键词 |
|------|------|--------|
| 1 | 年限 + 方向（让 HR 1 秒判断）| `8 年 Java + 1 年 AI` |
| 2 | 技术栈 + 当前焦点（ATS 关键词）| `Spring AI / RAG` |
| 3 | 标志性项目（面试官提问的钩子）| `200+ 文档 / 50+ QPS` |
| 4 | 擅长能力 + 量化（差异化）| `千万级 DAU` |
| 5 | 闭环能力（资深工程师硬通货）| `从需求到运维` |

### ⚠️ 红线

- ❌ "拥抱变化、追求技术、爱学习" —— 全是废话
- ❌ "对 AI 充满热情" —— 谁不充满热情
- ❌ 超过 6 行 —— HR 不看

## 三、技能栈（分层写法）

```
编程语言    ▎Java 17 (精通,8年) | Python 3.10 (熟练,1年) | SQL | Shell
后端框架    ▎Spring Boot 3 / Spring AI / Spring WebFlux / Spring Security / Spring Cloud Alibaba
AI 工程化   ▎大模型应用开发、RAG 架构、Prompt 工程、Function Calling、Agent 编排（ReAct/Plan）、MCP
向量 / 检索 ▎Milvus / pgvector / Elasticsearch (KNN) / 混合检索 (BM25 + Embedding + Reranker)
模型生态    ▎OpenAI / Claude / 通义千问 / DeepSeek / Ollama 本地部署
中间件      ▎MySQL / Redis / Kafka / RabbitMQ / Elasticsearch / MinIO
工程能力    ▎多租户架构、流式推理 (SSE/WebSocket)、限流熔断 (Resilience4j)、链路追踪、Prometheus + Grafana
```

### 🎯 4 个关键设计

1. **分层**：让人一眼看到能力图谱（语言 / 框架 / 业务方向 / 中间件）
2. **标深度**：在主语言后加 `(精通,8年)`，**坦诚 + 显得清晰**
3. **重点前置**：Java 在第 1 行，AI 在第 3 行（你的核心是"Java + AI"）
4. **AI 部分写"做的事"而不是"工具名"**：写"RAG 架构"比"LangChain4j"强

### 🚫 不写

- ❌ "熟悉计算机网络 / 数据结构" —— 8 年了，显初级
- ❌ "了解 Hadoop / Spark" —— 不是方向
- ❌ "熟悉 Vue / React" —— 除非强调全栈

## 四、工作经历 ⭐ 简历最重要的部分

### 4.1 写法模板

```
当前公司 ▎XX 部门 ▎职位 ▎2020.07 - 至今

【项目 1】企业 RAG 知识库系统 ▎主导设计 + 后端 lead ▎2025.10 - 至今  ⭐
─────────────────────────────────────────────────────
项目背景：解决 200+ 内部文档分散在 Confluence/SharePoint 难以检索的痛点，
        构建基于 Spring AI + Milvus 的智能问答系统。

技术栈：Spring Boot 3 + Spring AI 1.0 + Milvus + MySQL + Redis + RabbitMQ
       + 通义/GPT-4o/Ollama 多模型 + Docker Compose

核心职责（5-7 条，每条都带数字）：
1. 主导整体后端架构设计，基于 Spring Boot + Spring Security 构建多租户分层架构，
   实现 user_id 全链路隔离，支持 50+ 部门独立知识空间

2. 设计 MySQL + Redis + Milvus + MinIO 多级存储体系，
   实现元数据/向量/原文/对象 4 层分离，单租户文档容量从 1K → 10W+

3. 基于 RabbitMQ 解耦文档上传与向量化流程，
   500MB 文档入库从同步 120s → 异步 2s 返回 + 后台处理，QPS 提升 8x

4. 实现"向量召回 + BM25 + Reranker"三阶段混合检索，
   Top-3 召回率从 65% → 90%（基于 30 题人工评测集）

5. 基于 SSE + WebFlux 实现流式问答，
   首字延迟 P50: 1.8s → 0.6s，P95 < 800ms（50 QPS 压测）

6. 实现多模型路由（通义/GPT-4o/Ollama），按场景路由 + 失败降级，
   月度推理成本节省 ~40%（$680 → $410）

7. 设计 UsageRecorderAdvisor + Prometheus 监控，
   实时上报 token 用量 + 成本告警，超额自动限流

技术亮点：
✦ 父文档-子切片二级索引：用 256 token 小块做检索精度，
  用 1024 token 大块做生成上下文，元数据 ID 回溯
✦ Prompt 注入 5 道防线：输入清洗 + 角色锁定 + 工具白名单 +
  输出审查 + 限流，95% 攻击样本拦截
✦ 故障降级：通义/GPT-4o 全挂时自动切 Ollama 本地，
  从"全挂"变"降级运行"


【项目 2】订单中台微服务重构（AI 视角重写）▎主程 ▎2023.03 - 2024.06
─────────────────────────────────────────────────────
项目背景：原订单系统单体架构，QPS 上限 800，无法支撑业务增长。

核心职责：
1. 主导订单系统从单体演进到 Spring Cloud Alibaba 微服务，
   QPS: 800 → 5000，平均响应 230ms → 80ms

2. 设计基于 Redis + 分布式锁 + 消息补偿的最终一致性方案，
   替换原两阶段提交，吞吐提升 3x，零数据不一致事故运行 14 个月

3. 撰写《订单中台接入规范 v2》，3 个月内 4 个上游业务接入，
   沉淀复用率达 80%

⭐ 这个项目为后续 AI 能力接入提供高可用底座 —— 当前 RAG 知识库就部署在此架构之上


【项目 3】XX 支付网关 ▎核心后端 ▎2021.05 - 2023.02
─────────────────────────────────────────────────────
(2-3 行简写，省略细节)
```

### 4.2 ⭐⭐⭐ 写法心法

#### 心法 1：项目按价值倒序，AI 项目永远第一

```
项目 1 = 阶段 3 的 RAG 知识库（你的"灵魂项目"）  ← 占 50% 篇幅
项目 2 = 老 Java 项目（用 AI 视角重写）         ← 占 30%
项目 3 = 老 Java 项目（极简）                   ← 占 20%
```

#### 心法 2：每条职责都是 STAR + 数字

**公式**：`动词 + 技术方案 + 量化效果`

```
✅ "基于 RabbitMQ 解耦文档上传与向量化流程，QPS 提升 8x"
   = 动词(基于) + 技术(RabbitMQ 解耦) + 效果(8x)

❌ "用 RabbitMQ 做了异步处理"
   = 没动词 + 没技术取舍 + 没数字
```

#### 心法 3：数字越具体越可信

```
✅ "首字延迟 P50: 1.8s → 0.6s，P95 < 800ms（50 QPS 压测）"
❌ "性能大幅提升"
❌ "提升 50%"（没基线）
```

#### 心法 4：技术亮点单独成块（让面试官有话问）

技术亮点是**面试钩子** —— 面试官看到"父文档-子切片二级索引"，必问"为什么这么设计"。
你已经准备好了答案 → 面试主动权在你手上。

## 七、3 种简历重灾区 ❌

### 红区 1：空话连篇

```
❌ "热爱编程，追求技术卓越"
❌ "良好的团队协作精神"
❌ "学习能力强，乐于挑战"

→ 一句都不要写。HR 看了直接划走。
```

### 红区 2：堆热词

```
❌ "熟悉 Kafka、RocketMQ、RabbitMQ、Pulsar 等消息中间件
    熟悉 MySQL、Oracle、PostgreSQL、MongoDB 等数据库
    熟悉 Redis、Memcached 等缓存
    熟悉 Spring、SpringBoot、SpringCloud、Dubbo..."

→ 列得越多越显假。挑你最熟的 1-2 个，写深度。
```

### 红区 3：动词没量化

```
❌ "负责 XX 系统的开发与维护"
❌ "参与 XX 项目的需求分析与设计"
❌ "完成 XX 模块的性能优化"

→ "负责 / 参与 / 完成" 全是无效动词。
   要写"主导 / 设计 / 重构 + 数字效果"。
```

## 八、对应职级的写法差异

| 经验 | 关键词 | 主导词 |
|------|--------|--------|
| 0-3 年（P5/P6）| 参与 / 实现 / 完成 | 业务功能 + 工具使用 |
| 3-5 年（P6/P7）| 设计 / 优化 / 重构 | 模块架构 + 性能数字 |
| **5-8 年（P7/P8）** | **主导 / 推动 / 沉淀** | **系统架构 + 跨团队影响 + 规范** |
| 8+ 年（P8/P9）| 规划 / 引领 / 制定 | 技术战略 + 团队建设 + 业界发声 |

**你 8 年 Java + 1 年 AI** → 对标 P7/T5，用"主导 / 推动 / 沉淀"开头。
