### 关键叙事原则

把现有 7-8 年 Java 工作的项目，**重写为 "AI 应用平台地基" 的叙事**：

| ❌ 旧叙事 | ✅ AI 时代叙事 |
|--------------|----------------|
| 负责 XX 业务后端开发 | 主导 XX 微服务架构，**为后续 AI 应用提供高可用底座** |
| 优化 SQL，QPS 提升 30% | 优化数据查询性能，**支撑 RAG 系统的实时检索** |
| 设计 XX 系统架构 | 设计可扩展架构，**支持灵活接入大模型与 Agent** |

> 💡 **不撒谎，但要用 AI 时代的视角重新讲故事**。详细 5 个改写案例见 [`项目包装案例.md`](./简历/项目包装案例.md)。

### 答题黄金 5 步（每题都用这个结构）

```
0-5 min：澄清需求（问 5-8 个问题）⭐ 体现资深
5-8 min：容量估算（QPS / 存储 / 带宽）
8-18 min：整体架构（一边画一边讲）
18-30 min：关键设计深入（面试官挑 2-3 个展开）
30 min：演进路线（v1/v2/v3，体现规划力）
```

### Week 3 任务清单

- [ ] 每题准备好 5-6 个被追问的答案
- [ ] 找朋友模拟系统设计面 1-2 场（录音回放）
