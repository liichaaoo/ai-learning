## 一、投递渠道与策略 ⭐

### 1.1 渠道优先级（按 ROI 排序）

| 优先级 | 渠道 | 转化率（推算）| 备注 |
|--------|------|--------|------|
| ⭐⭐⭐⭐⭐ | **内部转岗 / 内推**（朋友/同事在目标公司）| 30-50% | 简历直达 hiring manager |
| ⭐⭐⭐⭐ | 脉脉直聊 HR / 技术 leader | 15-25% | 显得"主动 + 资深"|
| ⭐⭐⭐⭐ | 猎头（针对 P7+）| 10-20% | 头部猎头：数字华夏 / 大瀚 / 信熹达 |
| ⭐⭐⭐ | LinkedIn / Boss 直聊 | 5-15% | 国际化公司更好用 |
| ⭐⭐ | Boss 直接投 / 51 / 智联 | 2-8% | 海投，量大但匹配度差 |
| ⭐ | 公司官网 / 校招页 | 3-10% | 适合大厂正式岗 |

**结论**：**70% 精力放前 3 个渠道，30% 海投兜底**。

### 1.2 目标公司分级

```
S 级（最优）：腾讯内部转岗（保 P 级 + 保股票 + 工龄连续）
  目标 BG：TEG（混元）/ CSIG（云大模型 + AI 应用）/
          PCG（内容 AI / 阅文 AI 写作）/ IEG（游戏 AI）

A 级：大厂外跳（接受跳槽风险）
  字节 / 阿里 / 美团 / 京东 / 百度 / 网易 / 小米

B 级：AI 独角兽（薪资 + 期权）
  MiniMax / 智谱 / 月之暗面 / 零一万物 /
  阶跃星辰 / DeepSeek / 元象 / 商汤

C 级：传统行业 AI 化
  招行 / 平安 / 中信 / 国电 / 三一重工 AI 部门
```

### 1.3 投递节奏

```
Week 1 (Day 1-3)：投 S+A 级（每天 5-10 家），主精力
Week 1 (Day 4-7)：投 B 级 + 等回复
Week 2：跑面试 + 继续补投 A/B 级
Week 3-4：根据 offer 进展决定是否扩大或收缩

⚠️ 红线：
  - 不要"梭哈一家"，至少同时进 5-8 家流程
  - 不要"等 A 公司答复再投 B 公司"——等不起
  - 同时进 3 家终面 = 谈薪筹码
```

## 二、面试全流程 ⭐⭐

### 2.1 大厂典型流程（4-6 轮）

```
HR 初筛（15 min）→ 1 面（技术，60 min）→ 2 面（技术深入，60 min）
→ 3 面（部门 leader，60 min）→ 交叉面（其他部门或 HR，30 min）
→ HR 终面 / 谈薪（30-60 min）→ Offer
```

**周期**：腾讯/字节/阿里通常 2-4 周（HR 跟进慢的话拖到 6 周）。

### 2.2 各轮考察重点

| 轮次 | 考察 | 占比 | 重点准备 |
|------|------|------|---------|
| HR 初筛 | 沟通 + 简历真实性 | 20% | 自我介绍 + 离职原因 + 期望薪资 |
| 1 面（一线工程师）| 基础 + 算法 + 项目深挖 | 30% | 八股 + 1-2 道算法 + 项目 STAR |
| 2 面（资深工程师）| 系统设计 + 架构思维 | 30% | RAG/Agent 系统设计 + 取舍 |
| 3 面（leader）| 视野 + 业务理解 + 文化 | 10% | 行业认知 + 你想做什么 + 价值观 |
| 交叉面 | 反向校验 + 补漏 | 5% | 补充历轮的疑点 |
| HR 终面 | 薪资匹配 + 入职信息 | 5% | 谈薪策略 |

### 2.3 项目深挖：6 层追问怎么准备

> 面试官从你简历挑一条职责，会一直追问到你答不上。**提前准备到第 5-6 层**。

**例**：简历写"基于 RabbitMQ 解耦文档上传与向量化流程，QPS 提升 8x"

```
Lv1: RabbitMQ 解耦了什么？为什么要解耦？
  → 上传同步走 Embedding 太慢，用户等不了
  
Lv2: 为什么用 RabbitMQ 不用 Kafka？
  → 队列模型更简单，业务量级 RabbitMQ 够；Kafka 优势在大数据流
  
Lv3: 消息怎么保证不丢？
  → ack 模式 manual + 持久化 + DLQ 死信队列
  
Lv4: 消费者宕机消息会重复消费吗？
  → 会，所以要在业务层做幂等（task_id 唯一索引）
  
Lv5: 为什么 QPS 是 8x 不是 2x 不是 16x？
  → 实测：embedding 调用是同步阻塞，单机能并行 8 个 worker
       8x 是 worker 池调出来的最佳值
  
Lv6: 那你怎么调到 8 这个值的？
  → 压测脚本 + 1/4/8/16 实验，8 时 P99 最低，>8 反而下降（GC + 网络拥塞）
```

**自检**：你简历里 30 条职责，**每条都过一遍 6 层**，能答清楚再投。

### 2.4 系统设计面：30 分钟答题模板

参考 [`./系统设计/01-企业RAG系统.md`](./系统设计/01-企业RAG系统.md) 的"答题结构"：

```
0-5 min：澄清需求（问 5-8 个问题，体现资深）
5-8 min：容量估算（QPS / 存储 / 带宽）
8-18 min：整体架构（一边画一边讲）
18-30 min：关键设计深入（面试官挑 2-3 个让你展开）
30 min：演进路线（v1/v2/v3，体现规划力）
```

⚠️ **大忌**：上来就画图。资深的标志是**先问需求**。

### 2.5 算法面：要不要刷？

> 5+ 年简历 **算法占比下降** 但不能 0：
>
> - 中厂/独角兽：1 道中等题（Top 100 类）
> - 大厂（字节最严）：1 道中等 + 1 道困难，必刷
> - 腾讯：偏中等，DP / 贪心 / 二叉树高频
>
> **节奏**：阶段 5 第 1 周每天刷 3 题（LeetCode 热题 100），刷 50 道有信心了。

## 三、面试准备清单 ⭐

### 3.1 自我介绍（决定面试 30% 印象）

**3 分钟版本（技术面用）**：

```
您好，我是 XXX，8 年 Java 后端经验，主要在 [公司] 做 [核心业务]。

近一年专注 Spring AI + RAG + Agent 方向，主导了两个项目：

1. [企业 RAG 知识库] —— 200+ 内部文档，混合检索 + 多模型路由，
   召回率 65%→90%，月度成本节省 40%。

2. [AI Code Reviewer] —— 基于 LangChain4j 的 Multi-Agent 代码审查机器人，
   接入 GitHub Webhook，识别准确率 85%。

技术栈上 Java + Spring 生态是我的根，AI 方向我熟悉
Spring AI / LangChain4j / Milvus / 多模型路由 / Function Calling / MCP 协议。

我希望加入 [目标公司] 的 [目标岗位]，把过去的高并发后端架构经验
和 AI 应用结合，落地企业级 AI 产品。
```

**4 个关键设计**：
1. 第 1 句：定位（年限 + 方向）
2. 第 2-3 句：标志性项目 + 数字（让面试官有钩子提问）
3. 第 4 句：技术栈（关键词覆盖）
4. 第 5 句：求职动机（让 HR 觉得"匹配"）

### 3.2 离职原因（HR 必问 + 容易踩雷）

**禁忌**：
- ❌ 说前公司不好（"加班太多 / leader 不行 / 薪资低"）
- ❌ 太空泛（"想换个环境 / 寻求挑战"）

**推荐**：

```
"在 [前公司] 做了 N 年，从 [角色] 成长到 [角色]，
完整经历了 [项目] 从 0 到 1。最近一年公司战略偏向 [方向]，
我个人更想深入 AI 应用方向，所以想找一个 AI 业务更核心的团队。
[目标公司] 的 [部门] 在 [具体业务] 上的布局非常吸引我。"
```

**3 个要点**：
1. 肯定老东家（成长 / 经历）
2. 委婉说"业务方向不匹配"
3. 落到目标公司具体吸引点（**做过功课的标志**）

### 3.3 期望薪资（HR 初筛必问）

**报价策略**：

```
现状：当前总包 X
目标：X + 30%~50%（涨 30% 算保底，50% 算理想）

报价话术：
  "我现在总包是 [X]，加上 [年终/股票/福利] 大概 [X+1]。
   考虑到目标岗位偏 AI 方向是我职业转型的方向，
   我希望这次能在 [X*1.4 ~ X*1.5] 这个区间。
   具体我们可以根据 grade 和职级谈。"

⚠️ 不要："给多少都行" → HR 直接给最低
⚠️ 不要：报死数字 → 说"区间"留谈判空间
```

**多 offer 谈薪**：

```
"我目前还在面 [腾讯 / 字节 / 阿里] 的同级别岗位，
其中 [A 公司] 已经口头给了 [X*1.45]，但我个人对 [贵司业务] 更感兴趣，
希望薪资能保持竞争力。"
```

⚠️ **底线**：
- **不能虚报 offer**（HR 会反向核实）
- **真有 offer 才说**

### 3.4 反问环节（面试结尾必出现）

**4 个高质量反问**（按面试官类型挑 2-3 个）：

```
对一线工程师：
  Q1: 团队目前最大的技术挑战是什么？
  Q2: 团队的技术栈和迭代节奏？多久一次发版？
  Q3: 入职后前 3 个月主要会做什么？

对 leader：
  Q4: 团队成员主要来自哪些背景？团队组成？
  Q5: 团队未来 1 年的业务方向？AI 落地的具体规划？
  Q6: 您最近在思考的核心问题是什么？

对 HR：
  Q7: 公司福利体系？股票/期权？发布周期？
  Q8: 这个岗位 grade 范围？薪酬结构？
  Q9: 后续流程是？多久能给反馈？

⚠️ 不要问：
  - "996 吗"（直接显得你不愿加班）
  - "晋升怎么搞"（显得功利）
  - "加班多吗"（同上）
```

## 五、Offer 比较与决策 ⭐

### 5.1 总包计算公式

```
总包 = 月薪 × 12 + 月薪 × N（年终月数）
     + 股票/期权（按 4 年 vesting，每年到手 1/4）
     + 各种补贴（餐饮 / 交通 / 健身 / 房补）
     + 五险一金（含公积金可达月薪 24%）
```

**注意**：
- 股票/期权 **打 7 折**（流动性差 + 行权风险）
- 期权要看公司估值是否真实（独角兽 vs 上市公司差很多）
- N 是承诺还是实际？多问历史发了几个月

### 5.2 多 offer 决策矩阵

```
            权重  腾讯外跳  字节  独角兽A
─────────────────────────────────────────
现金薪资     30%   85       95     90
股票期权     20%   80       70     95（高风险）
团队/leader  15%   85       80     75
业务方向     20%   90       85     95
公司前景     10%   90       85     70
个人成长     5%    80       90     85
─────────────────────────────────────────
加权总分           85.5     85.5   87.0
```

**结论**：87.0 与 85.5 差不多，**最终还是看你"愿意承担多少风险"**。

### 5.3 决策建议

| 情况 | 推荐选 |
|------|--------|
| 求稳 + 有家庭 | 腾讯内部转岗 / 大厂上市公司 |
| 求增长 + 接受风险 | AI 独角兽（拼期权）|
| 想做最前沿 | 顶级独角兽（DeepSeek / Moonshot）|
| 离 P 级 / 总包临门一脚 | 谈到位的那一家 |

## 六、谈薪策略 ⭐⭐⭐

### 6.1 谈薪原则

```
1. 永远不要先报价 → 让 HR 先报
2. 报价后停顿 5 秒 → 给压力，让 HR 自己加
3. 报区间不报死数 → "X-Y" 而不是 "Y"
4. 涨幅锚定多 offer → 有筹码才谈得动
5. 谈三遍：再加 5%，再加 5%，再加 3%
```

### 6.2 谈薪话术模板

**第一回合 HR 给数**：

```
HR: "您这个岗位我们能给到月薪 35k，年终 3 个月，股票 60 万 4 年。"
你: 沉默 3 秒... "这个数字距离我的预期有些差距。我目前手上有 [B 公司]
     已经口头 offer 38k × 16 + 股票 80 万。我个人对贵司业务更有兴趣，
     但希望薪资能保持竞争力。"
```

**HR 反弹**：

```
HR: "我们 grade 范围只能到这里了..."
你: "我能理解 grade 限制。能不能从 [签字费 / 股票数量 / 年终月数] 上找补一下？
     比如年终从 3 个月加到 4 个月，或者股票加到 80 万。"
```

**最后一击**：

```
HR: "可以再加 5% 月薪 + 股票加到 75 万。"
你: "感谢努力。能不能签字费再加 5 万？这样我能立即决定接受贵司。"
```

### 6.3 8 个谈薪可以谈的维度

```
1. 月薪基数
2. 年终月数（13/14/15/16/18 都谈过）
3. 股票/期权数量
4. 股票 vesting 速度（4-1-1-1-1 vs 4-25%-25%-25%-25%）
5. 签字费（一次性，逃离上家股票/期权损失）
6. 入职时间（晚 1 个月入职可以拿满前家年终）
7. 职级（grade 上不去，加薪资）
8. 远程办公 / 弹性工作（你住远 → 谈这个）
```

### 6.4 不要的红线

- ❌ **接 offer 后毁约**（业内圈子小 + 拉黑名单）
- ❌ **同时签 2 个 offer**（一旦曝光职业生涯重创）
- ❌ **谈薪过程态度傲慢**（HR 会撤回 offer）
- ❌ **超过 1 周不回应 offer**（HR 觉得你不诚意）
- ❌ **入职后才发现"年终是浮动的"**（offer 谈定时就要白纸黑字）

### 6.5 Offer 谈完后必做的 4 件事

- [ ] **要 Offer Letter**（书面版，含薪资结构所有细节）
- [ ] **核对每一项数字**（月薪 / 月数 / 股票 / 签字费）
- [ ] **入职日期、试用期、违约金**（试用期最长 6 个月，违约金不能超培训费）
- [ ] **保留所有沟通记录**（微信 / 邮件存档，万一出问题作证据）

## 七、入职前 30 天 + 入职后 90 天

### 7.1 入职前

```
- 跟前公司平和告别（社保 / 公积金转移 / 工作交接）
- 跟新公司 HR 主动对接入职手续
- 提前学新公司的内部文化（看官方公众号 / 知乎搜员工分享）
- 调休几天，让自己满血进入新工作
```

### 7.2 入职后 90 天

```
Day 1-30：
  - 摸清团队/业务/工具链
  - 找 mentor + 跟每个组员聊一次
  - 主动接 1 个小任务建立信任

Day 31-60：
  - 接核心模块 / 子系统
  - 拿 1 次明确的产出（merge 一个 feature / 解一个 P0 / 写一份方案）

Day 61-90：
  - 转正 review
  - 跟 leader 对齐 Q3/Q4 目标
  - 输出 1 篇技术分享
```
