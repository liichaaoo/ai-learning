## 🧭 今天的核心问题

> Transformer 里 Attention 的公式是：
> $$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right) V$$

- Q、K、V 都是**矩阵**

- softmax 把相似度变成**注意力权重**
- 再乘以 V 得到"加权后的输出"

## 一、矩阵乘法（Matrix Multiplication）⭐⭐⭐

### 1.1 基本规则

$(m, n) \times (n, p) = (m, p)$

**中间两个数必须相等**，外面两个数就是结果的形状。

```
(2, 3) @ (3, 4) = (2, 4) ✅
(2, 3) @ (4, 3) = ❌ 报错（中间 3 ≠ 4）
(2, 3) @ (2, 3) = ❌ 报错（中间 3 ≠ 2）
```

### 1.1.1 为什么"中间两个数"必须相等？⭐

这是初学者最容易忽略、又最该想清楚的问题。两种角度理解：

#### 角度一：从计算定义看（点积要对齐）

矩阵乘法的核心规则是：

> **C 中第 i 行第 j 列的元素 = A 的第 i 行 与 B 的第 j 列 的点积**

$$C_{ij} = \sum_{k=1}^{n} A_{ik} \cdot B_{kj}$$

而点积要求两个向量**长度相同**：

- A 的"一行"长度 = A 的**列数**
- B 的"一列"长度 = B 的**行数**

所以必须：**A 的列数 = B 的行数**，否则一个长度 3 的向量没法和长度 2 的向量逐项相乘求和——对不齐，乘不起来。

举个反例，$A_{2\times 3} \times B_{2\times 3}$ 为什么不行：

- A 的第 1 行：`[a1, a2, a3]` —— 长度 3
- B 的第 1 列：`[b1, b4]` —— 长度 2（因为 B 只有 2 行）

要算点积时第 3 个数找不到搭档，运算就崩了。

#### 角度二：从线性变换看（变换要接得上）⭐⭐

矩阵代表**线性变换**，矩阵乘法代表**变换的复合**：

- $B_{n \times p}$：把 $p$ 维向量 → 映射到 $n$ 维空间
- $A_{m \times n}$：把 $n$ 维向量 → 映射到 $m$ 维空间

复合起来：

$$\mathbb{R}^p \xrightarrow{B} \mathbb{R}^{\mathbf{n}} \xrightarrow{A} \mathbb{R}^m$$

**B 的输出维度，必须等于 A 的输入维度**，两个变换才能"插得上"。这正是"中间两数相等"的几何含义。

> 💡 在 LLM 里，每个权重矩阵 $W_{n \times m}$ 就是一次"维度变换"。一连串矩阵乘法 = 一连串维度变换接力，**前一层的输出维度必须等于后一层的输入维度**，否则模型根本搭不起来。

#### 形状不匹配怎么办？两种常见救法

| 想算 | 直接算 | 救法 1：转置一个 | 救法 2：换顺序 |
|---|---|---|---|
| $A_{2\times 3} \times B_{2\times 3}$ | ❌ | $A \times B^T$ → $(2,2)$ ✅ | $B^T \times A$ → $(3,3)$ ✅ |

```python
A = np.random.randn(2, 3)
B = np.random.randn(2, 3)

# A @ B          # ❌ ValueError: shapes (2,3) and (2,3) not aligned
A @ B.T          # ✅ shape: (2, 2)
B.T @ A          # ✅ shape: (3, 3)
```

> 这就是为什么 Attention 要写成 $QK^T$ 而不是 $QK$ —— Q 和 K 都是 `(seq_len, d_k)` 的同形状矩阵，**必须把 K 转置一下**才能相乘。这不是数学家"故意为难"，是规则使然。

#### 一句话记忆

> **左矩阵的列 = 右矩阵的行**，本质是"行向量和列向量长度要一致才能做点积"，几何上是"前一个变换的输出维度，必须等于后一个变换的输入维度"。

#### ⚠️ 顺带一提：矩阵乘法**不满足交换律**

即使 $A \times B$ 能算，$B \times A$ 不一定能算；就算都能算，结果通常也不相等：

```python
A = np.random.randn(2, 3)
B = np.random.randn(3, 2)
A @ B    # shape: (2, 2)
B @ A    # shape: (3, 3)  ← 形状都不一样了
```

> 反向传播推导时这点特别容易踩坑：**矩阵乘法看顺序、看形状**。

---

### 1.2 手算一遍（非常重要）

$$A = \begin{bmatrix} 1 & 2 \\ 3 & 4 \end{bmatrix}, \quad B = \begin{bmatrix} 5 & 6 \\ 7 & 8 \end{bmatrix}$$

计算 $C = A \times B$：

**规则**：C 的第 i 行第 j 列 = A 的第 i 行 · B 的第 j 列（点积）

- $C_{0,0}$ = [1, 2] · [5, 7] = 1×5 + 2×7 = **19**
- $C_{0,1}$ = [1, 2] · [6, 8] = 1×6 + 2×8 = **22**
- $C_{1,0}$ = [3, 4] · [5, 7] = 3×5 + 4×7 = **43**
- $C_{1,1}$ = [3, 4] · [6, 8] = 3×6 + 4×8 = **50**

$$C = \begin{bmatrix} 19 & 22 \\ 43 & 50 \end{bmatrix}$$

### 1.3 Python 验证

```python
import numpy as np

A = np.array([[1, 2], [3, 4]])
B = np.array([[5, 6], [7, 8]])

print(A @ B)
# [[19 22]
#  [43 50]]

# 或者：np.matmul(A, B)、np.dot(A, B) 结果一样
```

### 1.4 Java 工程师视角

本质就是**嵌套循环**：

```java
// 伪代码
for (int i = 0; i < m; i++) {
    for (int j = 0; j < p; j++) {
        double sum = 0;
        for (int k = 0; k < n; k++) {
            sum += A[i][k] * B[k][j];
        }
        C[i][j] = sum;
    }
}
```

但 NumPy / PyTorch 会用 **SIMD + GPU** 加速，比你自己写快 1000 倍。**不要自己写**。

## 二、矩阵乘法的常见形状

### 2.1 向量 × 矩阵

```python
x = np.random.randn(1024)        # shape: (1024,)
W = np.random.randn(1024, 512)   # shape: (1024, 512)
y = x @ W                        # shape: (512,)
```

**LLM 含义**：一个 token 的 1024 维向量，通过权重矩阵 W 变成 512 维。这是神经网络"全连接层"的基本操作。

#### 2.1.1 结果是什么？形状怎么算？

把向量看成"1 行的矩阵"，套之前的规则：

$$\underbrace{x}_{1 \times n} \times \underbrace{W}_{n \times m} = \underbrace{y}_{1 \times m}$$

> **n 维向量 × (n, m) 矩阵 = m 维向量**。中间的 n 必须相等，结果向量的长度 = 矩阵的列数。

```
(3,) @ (3, 4) = (4,)        ✅ 3 维 → 4 维
(1024,) @ (1024, 512) = (512,)   ✅ 1024 维 → 512 维
(3,) @ (4, 5) = ❌  报错
```

**手算一遍**：

$$x = [1, 2, 3], \quad W = \begin{bmatrix} 1 & 2 \\ 3 & 4 \\ 5 & 6 \end{bmatrix}$$

- $y_0 = [1,2,3] \cdot [1,3,5] = 1+6+15 = \mathbf{22}$
- $y_1 = [1,2,3] \cdot [2,4,6] = 2+8+18 = \mathbf{28}$

得到 $y = [22, 28]$，是个 2 维向量（因为 W 有 2 列）。

#### 2.1.2 两种"看"的方式（都要会）⭐

**视角 A：行视角（一组点积）**

> $y_j$ = $x$ 与 $W$ **第 j 列**的点积

矩阵的每一列像一个"探测器"，向量和它做点积，看在这个方向上的"投影/相似度"。

**视角 B：列视角（行向量的加权求和）⭐⭐**

> $y = x_0 \cdot W_{0,:} + x_1 \cdot W_{1,:} + x_2 \cdot W_{2,:} + \dots$

也就是：**$y$ = 把 W 的每一行按 $x$ 的分量加权混合起来**。

用上面的例子验证：

$$y = 1 \cdot [1, 2] + 2 \cdot [3, 4] + 3 \cdot [5, 6] = [22, 28] \;\;✅$$

> 💡 这个视角在 **Embedding 查表** 和 **Attention 加权求和 V** 时特别清晰：`weights @ V` 就是"用 weights 这组系数把 V 的每一行加权混合"。

#### 2.1.3 关键问题：`(1024, 512)` 里的 `512` 到底代表什么？⭐

这是看 LLM 代码时**最高频要问自己**的问题。三个层面理解：

**① 数学层面**：512 = 输出向量的维度（W 的列数）

**② 几何层面**：512 = 把向量送到的"目标空间"维度

| W 的形状 | 几何含义 |
|---|---|
| `(1024, 1024)` | 维度不变，纯做"旋转/拉伸" |
| `(1024, 512)` | **降维**（信息压缩） |
| `(1024, 4096)` | **升维**（信息扩展） |

**③ 神经网络层面：512 = "这一层有多少个神经元"** ⭐⭐

```
输入向量 x (1024 维)
        │
        ▼
┌──────────────────────┐
│  全连接层  W          │  W.shape = (1024, 512)
│  1024 个输入  →       │
│  512 个神经元         │  每列 = 1 个神经元
└──────────────────────┘
        │
        ▼
输出向量 y (512 维)
```

记住这个对应关系：

> **`W.shape = (输入维度, 输出维度) = (上一层神经元数, 这一层神经元数)`**

PyTorch 里写法对照（注意 PyTorch 内部存的是 `W^T`，但概念一样）：

```python
import torch.nn as nn
layer = nn.Linear(1024, 512)   # 1024 个输入特征 → 512 个神经元
```

#### 2.1.4 LLM 里 `512` 这种数字常见出处

| 出现位置 | 数字代表什么 |
|---|---|
| Embedding 矩阵 `(vocab_size, d_model)` | 每个 token 用多少维表示 = **模型的"宽度"** |
| Attention QKV 投影 `(d_model, d_model)` | 隐藏维度 |
| 多头切分 `(d_model, n_heads × d_head)` | 例：512 = 8 头 × 64 维/头 |
| FFN 扩展层 `(d_model, 4 × d_model)` | 中间临时升维（一般 ×4） |
| FFN 收缩层 `(4 × d_model, d_model)` | 再降回原维度 |

真实模型的 `d_model`：

| 模型 | `d_model` |
|---|---|
| 教学版 / 小 Transformer | 128 ~ 512 |
| BERT-base / GPT-2 small | 768 |
| LLaMA-7B | 4096 |
| LLaMA-65B | 8192 |

> 所以 `(1024, 512)` 你完全可以脑补成"一个小型 Transformer 里把 1024 维特征压到 512 维的投影层"。

#### 2.1.5 ⚠️ 数学教材 vs 深度学习代码的写法差异

教材里很多时候是 $y = Wx$（**矩阵左乘列向量**）：

$$\underbrace{W}_{m \times n} \times \underbrace{x}_{n \times 1} = \underbrace{y}_{m \times 1}$$

但 NumPy / PyTorch 几乎全是 $y = xW$（**行向量右乘矩阵**）：

$$\underbrace{x}_{1 \times n} \times \underbrace{W}_{n \times m} = \underbrace{y}_{1 \times m}$$

| 风格 | 公式 | x 形状 | W 形状 |
|---|---|---|---|
| 数学教材 | $y = Wx$ | $(n, 1)$ 列向量 | $(m, n)$ |
| 深度学习代码 | $y = xW$ | $(n,)$ 行向量 | $(n, m)$ |

> **看代码时认准 `x @ W` 这种右乘写法**，别被教材的形状写法搞混。

### 2.2 批量：矩阵 × 矩阵

```python
X = np.random.randn(100, 1024)   # 100 个 token，每个 1024 维
W = np.random.randn(1024, 512)
Y = X @ W                        # shape: (100, 512)
```

**LLM 含义**：一批 token 同时过全连接层。**GPU 并行化的精髓**。

### 2.3 Attention 的 QK^T ⭐

```python
# 假设 batch=1, seq_len=10, d_k=64
Q = np.random.randn(10, 64)   # 10 个 token 的 Query
K = np.random.randn(10, 64)   # 10 个 token 的 Key

# QK^T
scores = Q @ K.T              # shape: (10, 10)
# 含义：scores[i, j] = 第 i 个 token 的 Query 和第 j 个 token 的 Key 的相似度
```

`scores` 就是**注意力矩阵**。每一行表示：这个 token 对其他所有 token 的"关注程度"。

> 💡 **深刻理解**：Attention 的 $QK^T$ 本质上就是**每对 token 之间算一次点积（相似度）**。
> 你 Day 1 学的点积，在这里派上了大用处。

## 三、线性变换（Linear Transformation）

### 直觉：矩阵 = 一种变换

给一个向量 $x$，乘一个矩阵 $W$，得到一个新向量 $y = Wx$。

**不同的矩阵 → 不同的变换**：
- 旋转矩阵 → 旋转向量
- 缩放矩阵 → 拉伸/压缩向量
- 投影矩阵 → 把高维投到低维

### LLM 里的"变换"

```python
# Embedding 层：把 token ID 变成向量（这是一个"查表"式变换）
emb = EmbeddingTable[token_id]  # shape: (1024,)

# 线性层（全连接层）：一个矩阵变换
y = emb @ W   # W.shape = (1024, 512)

# 多层堆叠：变换套变换
h1 = x @ W1
h2 = h1 @ W2
h3 = h2 @ W3
# 这就是神经网络
```

**一句话**：神经网络 = **多个矩阵乘法 + 非线性激活函数** 串起来。

## 四、特殊矩阵（听过就行，不深挖）

| 矩阵 | 形状 | 长啥样 | 作用 |
|------|-----|-------|-----|
| **单位矩阵 I** | $n \times n$ | 对角线全 1，其他全 0 | $IA = A$，像数字里的 1 |
| **零矩阵** | 任意 | 全 0 | $0 + A = A$ |
| **对角矩阵** | $n \times n$ | 只有对角线有值 | 表示"各维独立的缩放" |
| **对称矩阵** | $n \times n$ | $A^T = A$ | Attention 的某些矩阵会是对称的 |


## 五、广播（Broadcasting）⭐⭐

### 为什么需要广播

想象这个场景：

```python
X = np.random.randn(100, 1024)   # 100 条向量，每条 1024 维
bias = np.random.randn(1024)     # 一个 1024 维的偏置

# 你想让每一条向量都加上 bias，怎么写？
```

**C 语言/Java 思维**：

```python
for i in range(100):
    X[i] = X[i] + bias   # 累
```

**NumPy 思维**：

```python
X = X + bias   # 一行完事！NumPy 自动把 bias 广播到 100 行上
```

### 广播规则（简化版）

两个数组做运算时，如果形状不同：
1. 从后往前对齐形状
2. **缺失的维度用 1 补齐**
3. 如果某个维度一个是 1、另一个是 n → 把 1 的那个**复制 n 份**
4. 如果维度完全匹配不上 → 报错

### 常见广播案例

```python
# 案例 1：(100, 1024) + (1024,)  →  每行都加
X = np.random.randn(100, 1024)
b = np.random.randn(1024)
Y = X + b                     # ✅ shape: (100, 1024)

# 案例 2：(100, 1024) + (100, 1)  →  每行加一个不同的标量
scale = np.random.randn(100, 1)
Y = X * scale                 # ✅ 每一行乘不同的数

# 案例 3：(100, 1024) + (100,)   →  ❌ 不匹配（后面不对齐）
```

### 为啥你必须懂广播

LLM 代码里广播**无处不在**：
- 加位置编码（每个序列位置加一个向量）
- 加 bias
- LayerNorm / RMSNorm 的归一化参数
- mask 操作

看不懂广播 → 看 LLM 代码时寸步难行。

## 六、Attention 公式"带你走一遍"（Preview）

不用看懂细节，感受一下你已经学了多少：

```python
# 假设：batch=1（省略）, seq_len=10, d_k=64

Q = ...   # shape: (10, 64)  ← 线性变换得到的 Query
K = ...   # shape: (10, 64)  ← Key
V = ...   # shape: (10, 64)  ← Value

# 第 1 步：Q 和 K^T 相乘得到相似度矩阵
scores = Q @ K.T             # shape: (10, 10)

# 第 2 步：除以 sqrt(d_k) 缩放（防止值过大）
scores = scores / np.sqrt(64)

# 第 3 步：softmax 归一化成概率（明天 Day 3 讲）
weights = softmax(scores)    # shape: (10, 10)

# 第 4 步：用权重加权 V
output = weights @ V         # shape: (10, 64)
```

**你今天学完已经能看懂 3/4 了**：
- ✅ Q、K、V 是矩阵
- ✅ $QK^T$ 是矩阵乘法
- ✅ 矩阵形状
- ✅ 第 4 步 weights @ V 也是矩阵乘法
- ⏳ softmax 明天讲
