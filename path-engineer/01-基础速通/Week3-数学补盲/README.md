## 🚦 4 条重要提醒

### 1. 不要重新看大学教材

你大学学过的线代教材，用来从头学**会淹死你**。本周所有材料都是"面向 LLM"重新组织的。

### 2. 不要逃避公式

工程师看公式有本能抗拒 —— "我 Java 这么多年都不用这东西"。
但**AI 论文全是公式**，未来做 RAG/Agent 系统设计时，遇到 Attention 改造、损失函数调整，**绕不过去**。

本周的公式都是**每天用的**，不是抽象的。

### 3. 代码要跑，不要只看

每个概念下面的 Python 代码**动手敲 + 跑一遍**，比看 10 遍教程管用。

### 4. 遇到"推导"直接跳过

教程里看到任何"推导""证明"字眼，**跳过**。你要的是**结论 + 直觉**，不是数学能力。
