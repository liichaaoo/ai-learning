## 一、机器学习 vs 传统编程（Java 工程师最该先懂的事）

### 传统编程的思路

```
输入（数据）+ 程序（规则）  →  输出（答案）
```

比如你写一个"判断邮件是否是垃圾邮件"的程序：

```java
// 传统写法：你写死规则
public boolean isSpam(String email) {
    if (email.contains("中奖")) return true;
    if (email.contains("微信付款码")) return true;
    if (email.contains("点击链接")) return true;
    // ... 你写 100 条规则，还是漏掉各种变体
    return false;
}
```

问题：
- 规则写不完
- 坏人改个字"中﹒奖"你就失效了
- 业务变化快，规则维护成本爆炸

### 机器学习的思路

```
输入（数据）+ 输出（答案）  →  程序（规则）  ← 这是 ML 产出的东西
```

同一个问题：

```python
# ML 写法：你给它 10 万封标好的邮件，它自己学出规则
model = LogisticRegression()
model.fit(X_train, y_train)  # X=邮件内容 y=是否垃圾邮件

# 使用
is_spam = model.predict(["中奖通知：请点击..."])  # → True
```

**核心差异**：
| 维度 | 传统编程 | 机器学习 |
|------|---------|---------|
| **规则来源** | 程序员写 | 数据"喂"出来 |
| **适应变化** | 手动改代码 | 喂新数据重训 |
| **擅长场景** | 明确规则的任务 | 规则模糊但有数据的任务 |

### 🎯 什么场景该用 ML？什么场景别用？

| 场景 | 用不用 ML |
|------|----------|
| 把用户年龄 × 1.5 | ❌ 用 `age * 1.5` 就行，别上 ML |
| 判断一句话是正面还是负面 | ✅ 规则写不完，上 ML |
| 从身份证号提取出生日期 | ❌ 正则 5 分钟搞定 |
| 给用户推荐他可能喜欢的商品 | ✅ ML 专场 |
| 计算订单总价 | ❌ 纯逻辑 |
| 预测下个月用户会不会流失 | ✅ ML 专场 |

**记住**：**能用 if/else 的不要用 ML，能用 ML 的不要用 if/else**。

## 二、ML 的三大家族

### 🎯 2.1 监督学习（Supervised Learning）⭐ 最常用

**一句话**：**给数据配了"标准答案"**，让模型学会从输入猜答案。

**场景比喻**：学生做练习册，每道题都有答案，做错了对答案改正。

**两大子任务**：

| 子任务 | 输出类型 | 例子 |
|-------|---------|------|
| **分类**（Classification） | 离散类别 | 垃圾邮件（是/否）、图片是猫狗鸡 |
| **回归**（Regression） | 连续数值 | 房价预测、气温、股价 |

**Java 类比**：
- 分类 ≈ 返回 `enum` 或 `boolean`
- 回归 ≈ 返回 `double`

### 🎯 2.2 无监督学习（Unsupervised Learning）

**一句话**：**数据没标准答案，让模型自己发现规律**。

**场景比喻**：给你一堆硬币混一起，你没被告诉"哪个是 1 元哪个是 5 毛"，但你能按大小、颜色把它们分堆。

**三大子任务**：

| 子任务 | 作用 | 例子 |
|-------|-----|------|
| **聚类**（Clustering） | 把相似的放一起 | 用户分群、新闻分类 |
| **降维**（Dimensionality Reduction） | 把高维数据"压扁" | 把 100 维数据降到 2 维可视化 |
| **异常检测**（Anomaly Detection） | 找出不正常的 | 信用卡盗刷、服务器异常 |

### 🎯 2.3 强化学习（Reinforcement Learning）

**一句话**：**通过"奖励和惩罚"让 Agent 学会最优策略**。

**场景比喻**：训练小狗，做对动作给骨头（奖励），做错了没零食。

**本周不学**。但你要知道：
- AlphaGo 用的就是这个
- ChatGPT 的 RLHF（基于人类反馈的强化学习）也来自这
- 你未来如果进算法方向，会再学

## 三、ML 的 7 个核心术语（一次讲清）

这些术语本周会反复出现，先记住：

| 术语 | 英文 | 人话解释 |
|------|-----|---------|
| **特征** | Feature | 描述一个样本的属性（比如房屋的面积、楼层） |
| **标签** | Label / Target | 你要预测的答案（比如房价） |
| **样本** | Sample / Instance | 一条数据（一套房 = 一个样本） |
| **训练集** | Train set | 用来学习的数据 |
| **测试集** | Test set | 用来评估的数据（学完后考试） |
| **模型** | Model | 学到的"规则"本身 |
| **训练** | Training / Fit | 让模型从训练集学到规则的过程 |

**举例**（二手房数据）：

```
面积(㎡)  楼层  朝向  ...  →  价格(万)   ← 这叫一个【样本】
 80       3    南         450          
 120      8    东         680
 ...
 └───── 特征 (X) ─────┘    └ 标签 (y) ┘
```

**前 80% 数据** = 训练集 → 给模型学习
**后 20% 数据** = 测试集 → 考察学得好不好

## 四、一个 ML 项目的标准 7 步流程 ⭐

这是**你最该记的一张图**。以后看任何 ML 代码，都按这 7 步拆解：

```
┌────────────────────────────────────────────┐
│ 1. 定义问题（最容易被忽略的一步）              │
│    - 是分类还是回归？                         │
│    - 目标指标是什么？（准确率？F1？）         │
│    - 业务上能接受多少错误？                   │
├────────────────────────────────────────────┤
│ 2. 收集 & 理解数据                            │
│    - 有多少数据？分布怎样？                    │
│    - 有缺失值/异常值吗？                      │
├────────────────────────────────────────────┤
│ 3. 数据清洗 & 特征工程                         │
│    - 填缺失、去异常                          │
│    - 文本 → 向量（后面会讲）                   │
│    - 标准化（让不同量纲可比）                  │
├────────────────────────────────────────────┤
│ 4. 划分数据集                                 │
│    - train : val : test = 70:15:15          │
│    - 有时不分 val（用交叉验证代替）            │
├────────────────────────────────────────────┤
│ 5. 选模型 & 训练                              │
│    - 简单问题先从逻辑回归起步                  │
│    - 结构化数据 → XGBoost 通常很强            │
├────────────────────────────────────────────┤
│ 6. 评估 & 调优                                │
│    - 交叉验证看稳定性                         │
│    - 网格搜索调超参数                         │
│    - 检查过拟合                               │
├────────────────────────────────────────────┤
│ 7. 部署 & 监控                                │
│    - 导出模型                                 │
│    - 提供预测接口                             │
│    - 监控模型漂移（performance decay）        │
└────────────────────────────────────────────┘
```

### 💡 你要记住的"坑"顺序

很多新人踩的坑其实都是**流程没走对**：

- ❌ "我 model.fit() 了，准确率 99%！" → 没划分训练/测试集，在训练集上测的（典型自欺欺人）
- ❌ "我跑了 10 个模型对比" → 没做数据清洗，垃圾进垃圾出
- ❌ "模型上线了效果不行" → 没做数据漂移监控

**好工程师的 ML 流程，70% 时间花在第 2-3 步（数据），只有 10% 花在第 5 步（选模型）**。

## 五、最小可运行示例（看懂就行，不用打字）

下面这段代码就是"最短的 ML 程序"。看懂每一步对应上面 7 步流程的哪一步：

```python
# 1. 导入
from sklearn.datasets import load_iris
from sklearn.model_selection import train_test_split
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import accuracy_score

# 2. 收集数据（这里用自带的鸢尾花）
X, y = load_iris(return_X_y=True)
# X 是特征，shape=(150, 4)，150 朵花、每朵 4 个特征
# y 是标签，shape=(150,)，每朵花对应的类别 0/1/2

# 3-4. 划分数据集
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)

# 5. 选模型 & 训练
model = LogisticRegression(max_iter=200)
model.fit(X_train, y_train)  # ← 这一步在"学习"

# 6. 评估
y_pred = model.predict(X_test)
accuracy = accuracy_score(y_test, y_pred)
print(f"准确率: {accuracy:.2%}")  # 约 97%

# 7. 使用（"部署"的最简版）
new_flower = [[5.1, 3.5, 1.4, 0.2]]
prediction = model.predict(new_flower)
print(f"这朵花是第 {prediction[0]} 类")
```

**你要看懂**：
- 哪行对应"训练"，哪行对应"预测"
- 为什么要 `train_test_split`
- `model.fit()` 和 `model.predict()` 的分工
