## ⚠️ 一些"反直觉"的提醒

### 1. 不要纠结"为什么数学是这样"

```python
# 逻辑回归的公式：p = 1 / (1 + e^(-z))
```

Java 工程师看到会想"为什么是这个式子？凭什么是 e？"

**答案**：这是几十年的数学史和统计学推演出来的"最合适"的函数。
**你不需要懂推导，只要知道"它输出 0-1 之间的概率"**。

### 2. 不要打开 Kaggle 开始打比赛

Kaggle 打比赛会让你：
- 掉进"调参吃人"的陷阱（一个月没了）
- 学到的技能和 AI 应用开发关系不大
- 以为"ML 工程师都在调参"（错）

**你的战场在 LLM 应用**，不在 Kaggle。本周扫盲即可。

### 3. 不要自己实现算法

"用 numpy 从零写一个决策树"—— 教程和博客很多这种。
**直接跳过**。你学的是"会用 + 能讲清楚"，不是"会造轮子"。

### 4. sklearn 的 API 不用背

你是 Java 工程师，已经见过千百个框架 API。sklearn 也是一样：
- 三板斧：`model.fit(X, y)` → `model.predict(X)` → `model.score(X, y)`
- 其他用到再查文档就行
