# 🧮 算法每日一题（横向贯穿模块）

> 🎯 **目标**：用最小时间投入，保证面试算法不挂科。**不是去打 ACM，是去过面试。**
>
> ⏱️ **节奏**：每天 15~30 分钟，6 个月不间断（≈ 180 题）
>
> 🧭 **权重**：⭐⭐⭐（面试必考，但不是核心竞争力）
>
> 🔗 **定位**：横向模块，和阶段 01-06 **并行推进**，不打断主线

---

## 📌 核心原则（务必先看）

> **你是 7 年 Java 工程师，不是应届生。算法对你来说是"防守题"，不是"进攻题"。**

### 三条铁律

1. **不刷 Hard，除非冲大厂**
   - 80% 中端公司：Easy + Medium 就够
   - 字节/Meta/Google 例外，那时再加练 Hard

2. **不超 30 分钟，超时直接看题解**
   - 你的时间在 RAG / Agent 项目上更值钱
   - 算法的边际收益递减很快

3. **只刷高频，不追全量**
   - LeetCode Hot 100 + 剑指 Offer 75 + 公司高频 = ≈ 200 题封顶
   - 反复刷透 100 题 >> 草草过 500 题

### 80/20 时间分配

| 权重 | 内容 |
|------|------|
| **50%** | LeetCode Hot 100（按主题刷）|
| **25%** | 公司高频题（投递前突击）|
| **15%** | 系统设计 / 八股相关算法（LRU、限流、一致性 Hash 等）|
| **10%** | 周末复盘 + 错题重刷 |

## 🔧 工具与方法

### 推荐刷题平台

| 平台 | 用途 |
|------|------|
| [LeetCode 中国](https://leetcode.cn) | 主战场，Hot 100 / 面试经典 150 |
| [CodeTop](https://codetop.cc) | **公司高频题**（按公司/岗位过滤）⭐ Phase 4 必用 |
| [灵茶山艾府题单](https://leetcode.cn/u/endlesscheng/) | 进阶（暂时不用）|

### 模板化沉淀（重要）

每刷完一个主题，**必须沉淀一份模板**到 `模板/` 目录：

```
模板/
├── 二分查找四种写法.md
├── BFS层序遍历模板.md
├── 回溯三件套模板.md
├── 动态规划五步法.md
└── 滑动窗口模板.md
```

> 💡 面试紧张时大脑一片空白，模板就是你的肌肉记忆。

### 错题本

每道做错或卡壳的题，记录在 `错题本.md`：

```markdown
## [hot-100] 接雨水
- 难度：Hard
- 首刷日期：2026-06-15
- 卡点：双指针的边界更新条件没想清楚
- 关键 insight：left_max < right_max 时一定接 left_max - height[i]
- 复刷日期：2026-07-01 ✅ / 2026-08-01 ✅
```

## 🔥 防断更机制

> **算法最大的敌人不是难度，是断更。**

1. **打卡日历**（[`打卡日历.md`](./打卡日历.md)）：每天一行，断 3 天必须周末补
2. **最低保底**：再忙也要看一眼当天题目+题解（5 分钟），不算 0 输出
3. **周日复盘**：本周做了几题、错了几题、沉淀了什么模板
4. **和主项目联动**：写 RAG / Agent 卡壳时，切去做一题换脑子
