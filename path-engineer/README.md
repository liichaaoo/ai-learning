## 📐 设计哲学

### 倒金字塔学习（核心）

```
❌ 学院派（从底往上爬）：
  数学 → ML → DL → Transformer → LLM → 应用
                                         ↑
                                    12 个月后才到这里

✅ 工程派（从应用层往下探）：
  Spring AI → RAG → Agent → 够用的 LLM 原理 → 按需补数学
     ↑
  3 周后就能做 demo
```

### 80/20 时间分配

| 权重 | 内容 |
|------|------|
| **40%** | Spring AI / LangChain4j / RAG 实战 |
| **25%** | Agent / MCP / 工程化 |
| **15%** | LLM 基础原理（够用） |
| **10%** | Python 速通 + HuggingFace |
| **10%** | 经典 ML + 数学（速览） |

### 三大原则

1. **产出驱动**：每阶段必须有 demo 或可写简历的产出
2. **按需补原理**：做项目遇到问题再去查原理，不预先深学
3. **保留母语优势**：Java 是核心武器，Python 只学到能看懂
