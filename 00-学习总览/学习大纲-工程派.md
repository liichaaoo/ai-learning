## 🧭 写在最前：思维转变记录

### 从 v1 到 v2 的认知迭代

这份大纲是我学习路径的**第二个版本**。第一个版本（`学习大纲.md`）是一份"学院派"路径：
数学 → ML → 深度学习 → Transformer → 大模型核心 → 应用，**总计 12 个月**。

学到大概两周时，产生了几个关键疑问：

> 1. "花 2 个月学经典 ML，对我目标岗位到底有多大用？"
> 2. "手推反向传播对拿 offer 帮不帮得上忙？"
> 3. "Java 是不是已经没用了，必须转 Python？"
> 4. "12 个月太长，7 年工作经验再学一年还不如先跳槽？"

### 4 个关键认知转变

#### 🔄 转变 1：**从"学院派"到"工程派"**

| 之前 | 现在 |
|------|------|
| 像读研究生一样系统学 | 像做项目一样**以产出为目标**学 |
| 先把基础全打牢再做项目 | **第 1 个月就要能 demo**，原理按需补 |
| 每个阶段都求完整 | **80/20 法则**，80% 时间花在 20% 最值钱的技能 |

#### 🔄 转变 2：**从"扔掉 Java 学 Python"到"Java 是核心武器"**

原先以为 AI 必须 Python，想彻底转栈。实际调研后发现：
- 算法研究 = Python 100% 没错
- **企业 AI 应用 = Java/Go 正在快速上位**（Spring AI 起飞、LangChain4j 成熟）
- **"懂 AI 的 Java 资深工程师"是稀缺品种**

结论：Python 只学到能看懂代码（1 周），主战场仍在 Java。

#### 🔄 转变 3：**从"全面深入"到"够用就好"**

原理的作用不是拿来炫耀，而是：
- ✅ 能和算法同事对话
- ✅ 能判断技术方案是否靠谱
- ✅ 面试八股不露怯
- ❌ 不需要手推公式、自己实现 Transformer

**原理权重从 60% → 20%，工程权重从 40% → 80%。**

#### 🔄 转变 4：**从"转行者"到"升级者"**

最关键的思维转变。

7 年 Java 不是"要被抛弃的东西"，而是"**尚未装备 AI 能力的地基**"。
行业不是要淘汰 Java 工程师，是要淘汰**"只会 CRUD 的 Java 工程师"**。

**正确的自我定位**：
> 我不是从 0 开始学 AI 的转行者，
> 我是带着 7 年工程能力去"升级"成 AI 应用架构师的人。

### 为什么保留 v1 而不是删除

- 🪞 **自我记录**：这份学习之路本身就是有价值的记忆
- 🔁 **可能回切**：如果未来想走算法路径，v1 仍是好参考
- 📚 **给他人参考**：不同背景的人会选不同路径，v1 对某些人仍然最优

## 🎯 v2 的核心设计哲学

### 原则 1：**倒金字塔学习**

```
❌ v1 学院派（从底往上爬）：
  数学 → ML → DL → Transformer → LLM → 应用
                                         ↑
                                    12 个月后才到这里

✅ v2 工程派（从应用层往下探）：
  Spring AI → RAG → Agent → 够用的 LLM 原理 → 按需补数学
     ↑
  3 周后就能做 demo
```

### 原则 2：**80/20 分配**

| 权重 | 内容 |
|------|------|
| **40%** | Spring AI / LangChain4j / RAG 实战 |
| **25%** | Agent / MCP / 工程化 |
| **15%** | LLM 基础原理（够用） |
| **10%** | Python 速通 + HuggingFace |
| **10%** | 经典 ML + 数学（速览） |

### 原则 3：**产出驱动**

每个阶段必须有**可演示、可写简历**的产出，不然就是自嗨。

### 原则 4：**按需补原理**

**做项目遇到问题 → 查原理 → 学完回到项目**。
绝不"先把原理学完再做项目"——那样原理会忘光。

### 🏁 阶段 1：基础速通（3 周）

**目标**：能看懂 AI 代码 + 理解基本概念，**不求会写**。

**权重**：⭐⭐（仅为后续扫盲）

#### Week 1：Python 够用级（3-5 天）
- [ ] 语法速成（1-2 天）：基于 Java 对照着学
- [ ] 数据结构：list/dict/set/tuple
- [ ] 列表推导式、生成器、装饰器
- [ ] NumPy 基础（3 小时够看懂 AI 代码）
- ❌ 跳过：Django/Flask（你有 Spring）

📚 资源：[廖雪峰 Python 教程](https://www.liaoxuefeng.com/wiki/1016959663602400) 前 30%

#### Week 2：经典 ML 扫盲（不是深学）
- [ ] 理解监督/无监督/评估指标（你已有笔记）
- [ ] 过一遍已写好的鸢尾花 Notebook
- [ ] 知道有哪些算法存在即可（决策树、SVM、XGBoost）
- ❌ 跳过：Kaggle 打比赛、调参

#### Week 3：数学补盲（**只学 LLM 相关**）
- [ ] 线性代数：向量、矩阵、**向量相似度**（Embedding 要用）
- [ ] 概率：条件概率、贝叶斯（RAG 召回要懂）
- [ ] 微积分：导数、梯度（知道反向传播干嘛）
- ❌ 跳过：特征值分解、SVD 推导、凸优化

**里程碑**：能和算法同事聊 10 分钟不露怯 ✅

## 📐 学习方法论（特别重要）

### 每周节奏

- **工作日晚上**：1.5 小时 × 5 = 7.5 小时（看视频、读文档）
- **周末**：3 小时 × 2 = 6 小时（写代码、做项目）
- **周度总结**：周日晚 30 分钟写"本周学了什么，下周干嘛"

### 遇到不懂的 3 步法

```
1. 先查 Spring AI / LangChain4j 官方文档（30%）
        ↓ 不行
2. 问 AI 助手（Claude / GPT / CodeBuddy）（50%）
        ↓ 还不行
3. 查论文 / 源码 / 知乎（20%）
```

### 不要做的事

- ❌ **不要**从头到尾读一本 AI 大部头书（太慢）
- ❌ **不要**追热点不落地（每周新论文、每月新框架，追不完）
- ❌ **不要**学 PyTorch 深到能自己训模型（不是你的赛道）
- ❌ **不要**死磕数学推导（理解即可）
- ❌ **不要**迷信 Python（Java 是你的底牌）

### 必须做的事

- ✅ **每个阶段必须有 demo**（哪怕很糙）
- ✅ **每周写周报**（记录学习轨迹）
- ✅ **项目代码全部 Git 管理**
- ✅ **每月做一次技术分享**（给自己、给团队都行）
- ✅ **每周至少和一个业内人聊 30 分钟**（脉脉、KM、公众号私信）
