# AI 大模型岗位技能图谱（个人定制版）

> 📅 调研时间：2026 年 5 月
> 🧑 适用对象：**腾讯 7 年 Java 后端工程师，应用层背景，目标转 AI 方向且注重薪资**
> 🎯 核心策略：**保留 Java 母语 + 重点学 Spring AI / LangChain4j，做"Java + AI"复合型人才**

---

## 一、市场真相（2026 年最新数据）

### 关键趋势

> **纯业务型 Java 工程师需求正在萎缩，但「Java + AI」复合型人才被大厂疯抢。**

数据支撑：
- **字节跳动 2026 春招**：5000+ 岗位，AI 相关占近半，**优先后端开发经验**
- **百度**：4000+ offer，AI 占比 **90%+**
- **字节大模型应用架构专家**：年薪最高 **128 万**，明确说"对 Java 背景适配性极高"
- **猎聘真实 JD**：「JAVA 大模型应用开发工程师」明确要求 Spring AI + LangChain4j + RAG

### 你的位置

```
传统 Java 工程师（淘汰中 ⬇️）
       ↓
你（正在过渡）
       ↓
Java + AI 复合型工程师（疯抢 ⬆️）
       ↓
AI 应用架构师 / Tech Lead（150-300W）
```

## 二、为你筛选的 3 条最优路径

### 🥇 路径 A：**腾讯内部转岗 AI 应用 / 中台**（推荐指数 ⭐⭐⭐⭐⭐）

#### 为什么是它
- **保 P 级、保股票、保社保连续性**（外跳大概率丢一年股票）
- 腾讯内部 AI 项目井喷：**混元、元宝、ima、豆包对标产品都在招人**
- 走内推转岗，**简历直接进面**，不用海投
- 你是腾讯人，**信息不对称的红利只对你开放**

#### 目标 BG / 团队
| BG | 重点团队 | 备注 |
|----|---------|------|
| **TEG** | 混元应用平台、AI 中台 | 最对口你 Java 应用层 |
| **CSIG** | 腾讯云 LLM 平台、IMA、元宝 | 大量 Java 后端在招 |
| **PCG** | 内容 AI、视频号 AI、QQ AI 助手 | 应用层强 |
| **IEG** | AI 游戏中台、互动 AI | 业务有趣 |
| **WXG** | 微信 AI 应用 | 难进但稳定 |

#### 转岗操作步骤
```
本月：
  - 当前部门主动接 AI 相关任务（哪怕边角料）
  - KM 上写 1-2 篇 AI 学习/实践分享
  - 内部脉脉、KM 评论关注 AI 团队的人

第 2-3 个月：
  - 完成 1 个 Spring AI demo（开源到工蜂或 GitHub）
  - 联系目标 BG 的 leader（通过认识的人引荐）
  - HR 系统申请内部转岗

第 4-6 个月：
  - 面试 + 谈 offer
  - 腾讯转岗一般 2-3 周走完流程
```

#### 预期薪资
- 同 BG 内转：保 P 级，**薪资基本不变**
- 跨 BG 转：可能小降 5-10%（但 AI 涨薪机会更多，2 年回血）
- **P7 范围**：80-130W
- **P8 范围**：130-200W

> ⚠️ 注意：腾讯内部转岗有"半年内不能再转"的限制，要谨慎选定。

### 🥈 路径 B：**外跳到 AI 应用平台高级 / 资深岗**（推荐指数 ⭐⭐⭐⭐）

#### 为什么列第二
- 薪资天花板比内转高 20-40%
- 但风险高：AI 公司可能裁员、文化激进
- 适合**已经有 AI 项目积累**之后再考虑

### 🥉 路径 C：**AI 平台 / MLOps 工程师**（推荐指数 ⭐⭐⭐）

#### 适合什么样的你
- 你想要**工作内容和现在差异不大**（继续写后端、调度、平台）
- 不想 AI 知识学太深
- 求稳

#### 但为什么排第三
- **薪资天花板低于路径 A/B 10-20%**
- 平台岗成长性弱于应用层（应用层更接近业务和创新）

#### 目标
- 大厂的 AI 训练平台、模型管理平台、推理服务平台
- 工作内容：K8s + Volcano + Ray + 模型部署 + 监控告警

## 三、❌ 不建议你走的路径

| 路径 | 为什么不推荐 |
|------|-------------|
| 大模型算法（SFT/RLHF） | 7 年 Java + 没数学博士，**追不上算法人员**，3-5 年内难达匹配你现在的薪资水平 |
| LLM Infra（vLLM/CUDA） | 要补 C++/CUDA，对 Java 老兵**时间成本太高**，性价比低 |
| 多模态 / 研究员 | 博士门槛或同等论文，不切实际 |
| 初级 AI 应用工程师 | 薪资倒挂，你 7 年经验**绝不能降级** |

## 四、技能矩阵（精准对标你的目标）

### 必备技能（必须练到熟练）

| 技能 | 当前需要程度 | 学习时间 |
|------|------------|---------|
| **Java + Spring Boot** | ⭐⭐⭐⭐⭐ 你已会 ✅ | - |
| **Spring AI** | ⭐⭐⭐⭐⭐ 必学 | 1 周入门，2 周熟练 |
| **LangChain4j** | ⭐⭐⭐⭐ 重要 | 1 周 |
| **RAG 全流程** | ⭐⭐⭐⭐⭐ 必学 | 2 周 |
| **向量数据库**（Milvus/Qdrant/PgVector） | ⭐⭐⭐⭐ | 3 天 |
| **Function Calling / MCP** | ⭐⭐⭐⭐⭐ | 1 周 |
| **Python 速读** | ⭐⭐ 看懂就行 | 3 天 |
| **LLM 基础原理** | ⭐⭐⭐ 概念懂就行 | 1 周 |

### 加分技能（有时间补）

| 技能 | 优先级 | 学习时间 |
|------|-------|---------|
| **K8s + 模型服务部署** | ⭐⭐⭐ | 你应该已会 |
| **Prompt Engineering** | ⭐⭐⭐ | 边做边学 |
| **Agent 框架**（LangGraph/AutoGen） | ⭐⭐ | 1 周 |
| **微调 LoRA**（用现成工具） | ⭐⭐ | 选学 |
| **PyTorch 基础** | ⭐ 看懂代码 | 选学 |

### 不需要深入的技能 ❌

- ❌ TensorFlow（已过时）
- ❌ CUDA Kernel 编写
- ❌ 论文级数学（线代/概率会用就行，不需要推导）
- ❌ Django / Flask（你已会 Spring）
- ❌ 自己训练大模型（非你的赛道）

## 五、6 个月行动计划（资深工程师加速版）

> 你不是从 0 学起，**重点在"够用 + 能讲清楚 + 能落地"**

### 第 1 个月：夯实基础（你正在这个阶段）
- ✅ 数学基础：线性代数（已学）、概率统计、微积分（够用即可）
- ✅ 经典 ML：监督/无监督（你正在做）、决策树、集成学习
- ✅ Python 速通（**3-5 天搞定**，廖雪峰前 30%）
- 🎯 **里程碑**：能跑通 sklearn 经典案例 + 看懂 Python AI 代码

### 第 2 个月：深度学习快速入门
- ✅ 神经网络基础（MLP、反向传播）
- ✅ PyTorch 入门（不深，能调 API 即可）
- ✅ Transformer 架构（重点 Attention 机制）
- ✅ HuggingFace 三件套（Transformers、Datasets、Tokenizers）
- 🎯 **里程碑**：理解 GPT 是怎么生成文本的

### 第 3 个月：⭐ Spring AI 主战场
- ✅ Spring AI 全面学习（按 [SpringAI入门.md](../07-Java-AI路径/笔记/SpringAI入门.md)）
- ✅ LangChain4j 学习
- ✅ 项目 1：用 Spring AI 写一个智能客服 demo
- 🎯 **里程碑**：第一个能 demo 给同事看的 Java AI 应用

### 第 4 个月：RAG + Agent 实战
- ✅ Milvus / Qdrant 部署 + 使用
- ✅ Embedding 模型选型（bge / m3e / OpenAI）
- ✅ 文档处理（切分、清洗、metadata）
- ✅ Agent + Function Calling + MCP
- ✅ **项目 2**：企业 RAG 知识库（必做核心项目）
- 🎯 **里程碑**：项目部署上线，能写技术博客

### 第 5 个月：包装 + 内推
- ✅ 简历改造（项目突出 AI 应用平台架构能力）
- ✅ GitHub 整理（Star 数不重要，代码质量重要）
- ✅ KM 文章 / 公众号文章 1-2 篇
- ✅ 联系内部转岗 / 外部内推渠道
- 🎯 **里程碑**：开始拿到面试

### 第 6 个月：面试 + 拿 Offer
- ✅ 八股文（LLM 原理、RAG、Agent、Spring AI）
- ✅ 系统设计题（参考 `07-Java-AI路径/README.md` 第七节）
- ✅ 模拟面试 3-5 场
- 🎯 **里程碑**：拿到至少 2 个 offer 用来谈判

## 六、薪资目标（基于你 7 年腾讯经验合理预期）

> 假设你目前 P6 末或 P7（具体看你自己情况）

| 路径 | 预期最低 | 预期最高 | 现实期望 |

| 腾讯跨 BG 转 | -10% | +10% | **持平** |
| 大厂外跳（字节/阿里） | +20% | +50% | **+30%** |
| AI 独角兽 | +30% | +80% | **+40%（高风险）** |

- **保底线**：不能低于现在
- **目标涨幅**：30-50%
- **谈判筹码**：7 年 + 腾讯背景 + 已落地的 AI 项目

## 七、面试核心准备方向

### 必考（基础）
1. LLM 工作原理（Token、Embedding、生成过程）
2. Prompt Engineering 各种技巧（CoT、Few-shot、ReAct）
3. RAG 全流程 + 各环节优化
4. Function Calling vs MCP

### 高频（中级）
1. Spring AI 核心抽象（ChatClient、Advisor、VectorStore）
2. 流式响应实现（SSE + Reactor Flux）
3. 向量数据库选型与设计
4. LangChain4j Memory 实现机制

### 高级（系统设计）
1. 设计企业级 RAG 系统（百万文档 + 万级 QPS）
2. 设计多模型路由网关
3. 设计 AI 客服系统的高可用方案
4. 设计 Token 成本控制方案

## 九、最后一段话

你的核心优势是：**7 年腾讯 + Java 应用层 + 注重薪资**。

最优解 **不是** "扔掉 Java 学 Python 从 0 开始"，
**而是** "保留 Java 主战场 + 用 Spring AI 升级武器"。

行业天天变，**底层逻辑（工程能力 + 资深资历 + 业务理解）** 不变。
你只需要补一层 AI 能力的"保护壳"，市场就会给你超额溢价。

> **6 个月内拿到 30%+ 涨薪的 offer**，这是基于市场数据的合理目标。
