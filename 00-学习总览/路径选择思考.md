# 路径选择思考 · 从学院派到工程派的认知迭代

> 📅 写于：2026 年 5 月 11 日
> 📌 这是一份**思维过程记录**，不是学习计划本身
> 🪞 留存目的：未来 3-5 年回看，会成为最珍贵的成长样本

---

## 背景

仓库初建时（2026-05-07），我设定的学习路径是**学院派**：

```
数学 → ML → 深度学习 → Transformer → 大模型核心 → 应用
（总计 12 个月）
```

学到大约两周时（线性代数练习 + 监督/无监督学习笔记 + 鸢尾花 Notebook 完成时），
产生了几个关键疑问：

> 1. "花 2 个月学经典 ML，对我目标岗位到底有多大用？"
> 2. "手推反向传播对拿 offer 帮不帮得上忙？"
> 3. "Java 是不是已经没用了，必须转 Python？"
> 4. "12 个月太长，7 年工作经验再学一年还不如先跳槽？"

调研了 2026 年的 AI 招聘市场后，做出**重大路径调整**。

---

## 4 个关键认知转变

### 🔄 转变 1：从"学院派"到"工程派"

| 之前的我 | 现在的我 |
|---------|---------|
| 像读研究生一样系统学 | 像做项目一样**以产出为目标**学 |
| 先把基础全打牢再做项目 | **第 1 个月就要能 demo**，原理按需补 |
| 每个阶段都求完整 | **80/20 法则**，80% 时间花在 20% 最值钱的技能 |
| 12 个月规划 | **6 个月冲 offer** |

---

### 🔄 转变 2：从"扔掉 Java 学 Python"到"Java 是核心武器"

最初以为做 AI 必须全身心转 Python，想彻底转栈。
实际调研后发现：

- 算法研究 = Python 100% 没错
- **企业 AI 应用 = Java/Go 正在快速上位**（Spring AI 起飞、LangChain4j 成熟）
- **"懂 AI 的 Java 资深工程师"是稀缺品种**

具体数据（2026-05 实时招聘市场）：
- 字节跳动 2026 春招 5000+ 岗位，AI 占近半，**优先后端经验**
- 字节"大模型应用架构专家"年薪最高 **128 万**，明说"对 Java 背景适配性极高"
- 真实 JD（猎聘）：「JAVA 大模型应用开发工程师」明确要求 Spring AI + LangChain4j + RAG
- 行业明确信号：**纯 CRUD 的 Java 在萎缩，但「Java + AI」复合型人才被疯抢**

**结论**：Python 只学到能看懂代码（1 周），主战场仍在 Java。

---

### 🔄 转变 3：从"全面深入"到"够用就好"

原理（数学、深度学习、Transformer 内部机制）的作用不是拿来炫耀，而是：

- ✅ 能和算法同事对话
- ✅ 能判断技术方案是否靠谱
- ✅ 面试八股不露怯
- ❌ 不需要手推公式
- ❌ 不需要自己实现 Transformer
- ❌ 不需要训练大模型

**原理权重从 60% → 20%，工程权重从 40% → 80%。**

不是"原理不重要"，而是**对我的目标岗位**（Java + AI 应用开发），原理够用就好。

---

### 🔄 转变 4：从"转行者"到"升级者"

最关键的思维转变。

之前的心态：**"我要扔掉 7 年 Java，从 0 开始学 AI"**
转变后的心态：**"我带着 7 年工程能力去给自己装上 AI 能力"**

7 年 Java 不是"要被抛弃的东西"，而是：

> **尚未装备 AI 能力的地基**

行业不是要淘汰 Java 工程师，是要淘汰**"只会 CRUD 的 Java 工程师"**。

**正确的自我定位**：
> 我不是从 0 开始学 AI 的转行者，
> 我是带着 7 年工程能力去"升级"成 AI 应用架构师的人。

## 决策的元思考

### 1. 决策不是选答案，是选问题

之前花了很多时间纠结"学院派 vs 工程派"，其实两条路没有对错，只有"是否匹配当前目标"：

- 应届生 / 5 年时间预算 / 想做研究 → 学院派对
- 7 年资深 / 想变现 / 要保持职业曲线上升 → 工程派对

**问对问题，答案就清晰了。**

### 2. 关于时间窗口

2026 年是一个特殊时间点：
- 大模型技术栈快速收敛
- 企业 AI 应用刚开始规模化
- 懂 AI 的工程老兵**稀缺**

**这个窗口最多持续 2-3 年**。3 年后 AI 应用技能会像今天的"会用 Git"一样普及，
那时候大家都懂 Spring AI，就没差异化了。

**所以现在是学 + 跳的最佳窗口。**

### 3. 关于"沉没成本"

已经写了的线代笔记、ML 笔记、Notebook —— **不是"白费"**，而是：

- 让我对 AI 不再陌生（认知红利）
- 是与算法同事对话的基础（社交红利）
- 转工程派后这些仍然是参考资料（资产红利）

**所以保留 path-research/，不删。**

### 4. 给未来自己的承诺

无论 6 个月后结果如何，我都要：
- 每周写周报，哪怕当周很失败
- 每个项目开源一份代码
- 半年后回来对比这份文档，看自己变了多少
